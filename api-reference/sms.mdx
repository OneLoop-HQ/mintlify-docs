## Overview

The SMS API lets you send outbound text messages from registered phone numbers, configure global SMS settings such as allowed destination countries and message length caps, and handle inbound messages through channel bindings on your registered phone numbers.

Authentication requires an `x-api-key` header on every request.

---

## Send an SMS Message

<Warning>
  Message bodies are truncated or rejected when they exceed `max_message_length`. Fetch your current settings with `GET /v1/sms/settings` before sending long messages.
</Warning>

```
POST /v1/sms/send
```

Sends an outbound SMS from a registered Feather phone number to a destination number. If you supply an `idempotency_key`, resubmitting the same key within 24 hours returns the original response without creating a duplicate message.

<Note>
  Always pass an `idempotency_key` when building retry logic. This prevents duplicate messages if your client times out before receiving a response.
</Note>

### Request Body

<ParamField body="from_number" type="string" required>
  The sender's phone number in E.164 format (e.g. `+14155550100`). Must be a phone number registered in your Feather account.
</ParamField>

<ParamField body="to" type="string" required>
  Destination phone number in E.164 format (e.g. `+447911123456`). Must belong to a country in your `allowed_countries` list.
</ParamField>

<ParamField body="body" type="string" required>
  The text content of the message. Must not exceed `max_message_length` characters.
</ParamField>

<ParamField body="agent_id" type="string (uuid)">
  The agent that will own this SMS session. If omitted, the default agent bound to `from_number` is used.
</ParamField>

<ParamField body="context_variables" type="object">
  Arbitrary key/value pairs injected into the session context and available to the agent during the conversation.
</ParamField>

<ParamField body="media_urls" type="array of strings">
  Public URLs of media attachments to include in the message (MMS). Supported types depend on the carrier.
</ParamField>

<ParamField body="idempotency_key" type="string">
  A client-generated unique key (UUID or any string up to 255 characters). Requests with the same key within 24 hours return the cached response.
</ParamField>

### Response

<ResponseField name="session_id" type="string (uuid)">
  The Feather session created or resumed for this SMS thread.
</ResponseField>

<ResponseField name="end_user_id" type="string (uuid)">
  The end-user record associated with the `to` number.
</ResponseField>

<ResponseField name="idempotency_key" type="string">
  Echoes the key you supplied, or a system-generated key if none was provided.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/sms/send \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "from_number": "+14155550100",
    "to": "+447911123456",
    "body": "Hi! Your support ticket #4821 has been updated.",
    "agent_id": "a1b2c3d4-0000-0000-0000-000000000001",
    "context_variables": {
      "ticket_id": "4821",
      "priority": "high"
    },
    "idempotency_key": "send-4821-2024-06-01T10:00:00Z"
  }'
```

```json Response
{
  "session_id": "sess_9f3e2a1b-0000-0000-0000-000000000042",
  "end_user_id": "usr_4c7d8e2f-0000-0000-0000-000000000099",
  "idempotency_key": "send-4821-2024-06-01T10:00:00Z"
}
```
</CodeGroup>

## Get SMS Settings

```
GET /v1/sms/settings
```

Returns the current organization-wide SMS configuration.

### Response

<ResponseField name="allowed_countries" type="array of strings">
  ISO 3166-1 alpha-2 country codes to which outbound SMS is permitted (e.g. `["US", "GB", "CA"]`). Messages to numbers outside this list are rejected before dispatch.
</ResponseField>

<ResponseField name="max_message_length" type="integer">
  Maximum number of characters allowed per message body. Between 1 and 1600.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/sms/settings \
  -H "x-api-key: <key>"
```

```json Response
{
  "allowed_countries": ["US", "CA", "GB", "AU"],
  "max_message_length": 1600
}
```
</CodeGroup>

## Update SMS Settings

```
PUT /v1/sms/settings
```

Replaces the organization-wide SMS configuration. All fields in the request body are required; omitted fields revert to platform defaults.

### Request Body

<ParamField body="allowed_countries" type="array of strings" required>
  ISO 3166-1 alpha-2 country codes to whitelist. Sending to any country not in this list will be blocked.
</ParamField>

<ParamField body="max_message_length" type="integer" required>
  Maximum message body length. Must be between **1** and **1600** inclusive. Encoding counts vary by character set — GSM-7 supports 1600 characters; messages with non-GSM characters may segment earlier.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X PUT https://api-sandbox.featherhq.com/v1/sms/settings \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "allowed_countries": ["US", "CA", "GB"],
    "max_message_length": 320
  }'
```

```json Response
{
  "allowed_countries": ["US", "CA", "GB"],
  "max_message_length": 320
}
```
</CodeGroup>

## Inbound SMS

Inbound messages are received automatically when a phone number has an `sms` channel binding. Feather creates a session for each new inbound thread and routes it to the bound agent or team. To configure bindings, use the [Phone Numbers API](/api-reference/phone-numbers).

<Tip>
  To test inbound SMS in the sandbox, send a message to your registered sandbox number from any E.164-formatted number. A session will be created and visible in the Sessions dashboard immediately.
</Tip>
