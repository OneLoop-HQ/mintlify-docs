## Overview

A scenario is a reusable test case that describes what a simulated end-user wants to accomplish, how they'll behave, what the world looks like (mock tool responses), and what success looks like. Scenarios can be authored manually, generated by AI from a brief, or drafted automatically from real conversations.

---

## Create a Scenario

```
POST /v1/scenarios
```

### Request Body

<ParamField body="name" type="string" required>
  Short, descriptive name for the scenario, e.g. `Refund request — happy path`.
</ParamField>

<ParamField body="intent" type="string" required>
  The end-user's goal, written in plain language. The simulated persona will pursue this intent during the conversation.
</ParamField>

<ParamField body="journey" type="string">
  A longer narrative describing the user's situation, emotional context, and any relevant backstory. Optional but improves simulation realism.
</ParamField>

<ParamField body="mock_world" type="object">
  A map of tool names to [CannedToolResponse](/api-reference/simulations/runs#cannedtoolresponse) objects. Use this to control what tool calls return during the simulation without hitting real external systems.

  ```json
  {
    "get_order_status": { "output": { "status": "shipped", "eta": "2024-06-05" } },
    "process_refund": { "error": "Refund service unavailable", "delay_ms": 200 }
  }
  ```
</ParamField>

<ParamField body="source" type="string">
  Origin of the scenario: `manual` (human-authored) or `drafted` (AI-generated). Default: `manual`.
</ParamField>

<ParamField body="source_agent_id" type="string (uuid)">
  The agent used to draft this scenario (if `source=drafted`).
</ParamField>

<ParamField body="source_persona_id" type="string (uuid)">
  The persona used to draft this scenario.
</ParamField>

<ParamField body="tags" type="array of strings">
  Labels for organizing scenarios (e.g. `["billing", "happy-path", "edge-case"]`).
</ParamField>

<ParamField body="agent_ids" type="array of strings (uuid)">
  Agents this scenario is associated with. Scenarios can be shared across multiple agents.
</ParamField>

<ParamField body="success_criteria" type="array of objects">
  Evaluation criteria that define what a successful simulation looks like. Each entry:
  - `description` (string, required) — Natural language description of the success condition.
  - `output_type` (string) — `boolean` (pass/fail) or `score_5` (1–5 rating).
</ParamField>

### Response

Returns a [ScenarioResponse](#scenarioresponse).

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/scenarios \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Refund request — order not received",
    "intent": "Get a refund for an order that was never delivered.",
    "journey": "The user ordered a laptop 3 weeks ago. Tracking shows it was delivered but they never received it. They are frustrated and want a refund immediately.",
    "mock_world": {
      "get_order_status": {
        "output": { "status": "delivered", "tracking": "1Z999AA10123456784" }
      }
    },
    "tags": ["billing", "refund", "edge-case"],
    "agent_ids": ["a1b2c3d4-0000-0000-0000-000000000001"],
    "success_criteria": [
      {
        "description": "Agent initiates the refund process without asking the user to call back.",
        "output_type": "boolean"
      },
      {
        "description": "Agent maintains an empathetic and professional tone throughout.",
        "output_type": "score_5"
      }
    ]
  }'
```
</CodeGroup>

## List Scenarios

```
GET /v1/scenarios
```

### Query Parameters

<ParamField query="source" type="string">
  Filter by `manual` or `drafted`.
</ParamField>

<ParamField query="source_agent_id" type="string">
  Filter to scenarios drafted by a specific agent.
</ParamField>

<ParamField query="source_persona_id" type="string">
  Filter to scenarios drafted with a specific persona.
</ParamField>

<ParamField query="agent_id" type="string">
  Filter to scenarios associated with a specific agent.
</ParamField>

<ParamField query="tags" type="array of strings">
  Filter to scenarios that have all of the specified tags.
</ParamField>

<ParamField query="created_after" type="string (ISO 8601)">
  Return only scenarios created after this timestamp.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor.
</ParamField>

## Draft a Scenario with AI

```
POST /v1/scenarios/draft
```

Uses AI to generate a scenario from a brief description. The scenario is **not saved** — inspect the draft and call `POST /v1/scenarios` to persist it.

### Request Body

<ParamField body="brief" type="string" required>
  A short description of the scenario you want to create, e.g. `"A customer asking about a delayed shipment during peak holiday season"`.
</ParamField>

<ParamField body="agent_id" type="string (uuid)" required>
  The agent context to draft the scenario for. The agent's capabilities and knowledge inform the generated scenario.
</ParamField>

<ParamField body="persona_id" type="string (uuid)">
  A persona to simulate in the drafted scenario.
</ParamField>

### Response

Returns a `ScenarioDraft` object with all fields populated but no `id` (it has not been saved). Review and adjust before persisting.

---

## Draft a Scenario from a Session

```
POST /v1/scenarios/draft-from-session
```

Automatically generates a scenario by analyzing the turns in an existing real conversation. The result captures the user's actual intent and journey.

### Request Body

<ParamField body="session_id" type="string (uuid)" required>
  The session to generate the scenario from.
</ParamField>

### Response

Returns a `ScenarioDraft` object. Review and persist with `POST /v1/scenarios`.

<Tip>
  Use `draft-from-session` to quickly build a regression test suite from interesting or problematic conversations in production.
</Tip>

</ResponseField>

</ResponseField>
