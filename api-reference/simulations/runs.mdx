## Overview

Individual simulation runs are created either as part of a [suite run](/api-reference/simulations/suites) or directly via `POST /v1/simulation/runs`. Each run produces a conversation transcript, eval results, and a detailed execution trace you can use to understand exactly how the agent reasoned through the scenario.

---

## Create a Run

```
POST /v1/simulation/runs
```

Creates and immediately dispatches a single simulation run. Use this for ad-hoc testing outside of a suite.

### Request Body

<ParamField body="agent_id" type="string (uuid)" required>
  The agent to simulate against.
</ParamField>

<ParamField body="agent_revision_id" type="string (uuid)">
  Pin the run to a specific agent revision.
</ParamField>

<ParamField body="persona_id" type="string (uuid)" required>
  The persona to simulate as the end-user.
</ParamField>

<ParamField body="scenario_id" type="string (uuid)" required>
  The scenario defining the user's intent and mock world.
</ParamField>

<ParamField body="channel" type="string">
  `text` or `voice`. Default: `text`.
</ParamField>

<ParamField body="tool_overrides" type="object">
  Map of tool names to [CannedToolResponse](#cannedtoolresponse) objects. Overrides scenario-level `mock_world` for this run only.
</ParamField>

## List Runs

```
GET /v1/simulation/runs
```

### Query Parameters

<ParamField query="status" type="string">
  Filter by run status: `pending`, `running`, `completed`, `failed`, `cancelled`.
</ParamField>

<ParamField query="agent_id" type="string">
  Filter to runs for a specific agent.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor.
</ParamField>

## Get a Run

```
GET /v1/simulation/runs/{id}
```

Returns the full simulation run record.

### SimulationRunResponse

<ResponseField name="id" type="string">
  Unique run identifier.
</ResponseField>

<ResponseField name="session_id" type="string">
  The Feather session created for this simulation.
</ResponseField>

<ResponseField name="agent_id" type="string">
  Agent that handled the simulation.
</ResponseField>

<ResponseField name="persona_id" type="string">
  Persona used as the simulated end-user.
</ResponseField>

<ResponseField name="scenario_id" type="string">
  Scenario the run was based on.
</ResponseField>

<ResponseField name="channel" type="string">
  `text` or `voice`.
</ResponseField>

<ResponseField name="status" type="string">
  `pending`, `running`, `completed`, `failed`, or `cancelled`.
</ResponseField>

<ResponseField name="started_at" type="string (ISO 8601)">
  When the simulation started.
</ResponseField>

<ResponseField name="finished_at" type="string (ISO 8601)">
  When the simulation finished. `null` if still running.
</ResponseField>

<ResponseField name="error_text" type="string">
  Error message if `status` is `failed`. `null` otherwise.
</ResponseField>

<ResponseField name="recording_id" type="string">
  ID of the associated voice recording, if `channel=voice`.
</ResponseField>

## Get Run Trace

```
GET /v1/simulation/runs/{id}/trace
```

Returns the full paginated execution trace for a simulation run. The trace captures every step the agent took — model calls, tool invocations, memory reads, policy checks, and persona responses.

### Trace Step Types

<ResponseField name="CustomerAgent" type="object">
  A step where the simulated persona (customer) sent a message. Contains the persona's message text.
</ResponseField>

<ResponseField name="System" type="object">
  An internal system step: model reasoning, tool call, tool result, policy evaluation, or handoff. Contains `step_type`, `content`, and `latency_ms`.
</ResponseField>

### Query Parameters

<ParamField query="cursor" type="string">
  Pagination cursor for trace steps.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/simulation/runs/simrun_aabbccdd-0000-0000-0000-000000000091/trace" \
  -H "x-api-key: <key>"
```

```json Response
{
  "steps": [
    {
      "type": "CustomerAgent",
      "sequence": 1,
      "content": "I never received my order and I want a refund immediately.",
      "timestamp": "2024-06-01T14:00:01Z"
```

## Get Turn Trace

```
GET /v1/simulation/runs/{id}/turns/{turn_id}/trace
```

Returns the trace steps scoped to a single conversation turn within the simulation. Useful for drilling into a specific agent response without loading the entire run trace.

## CannedToolResponse

Used in `tool_overrides` on both suite runs and individual runs to control tool call behavior during simulations.

<ParamField body="output" type="object">
  The mock output to return from the tool call. Either `output` or `error` must be set.
</ParamField>

<ParamField body="error" type="string">
  An error message to return instead of a successful output. Either `output` or `error` must be set.
</ParamField>

<ParamField body="delay_ms" type="integer">
  Optional artificial delay in milliseconds before returning the response. Useful for testing timeout-handling behavior.
</ParamField>

### Example

```json
{
  "get_order_status": {
    "output": { "status": "shipped", "eta": "2024-06-05" },
    "delay_ms": 150
  },
  "process_refund": {
    "error": "Refund service temporarily unavailable"
  }
}
```

<Note>
  `delay_ms` adds wall-clock latency to the simulation. If your agent has a tool timeout policy, setting `delay_ms` above the timeout threshold is a useful way to test timeout-handling paths.
</Note>
