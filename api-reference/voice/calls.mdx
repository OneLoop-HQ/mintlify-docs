## Overview

Feather's voice call APIs cover three call scenarios:

- **Outbound** — Feather dials a phone number on behalf of your agent
- **Web call** — Browser-based audio call via LiveKit for testing and embedded UI use cases
- **Ingest** — Upload a pre-recorded call audio file and metadata for processing and transcription

**Base URL:** `https://api-sandbox.featherhq.com`

---

## Outbound Calls

### Place an Outbound Call

```
POST /v1/voice/outbound
```

Instructs Feather to dial a phone number and connect the call to an agent.

<ParamField body="from_phone_number_id" type="string" required>
  The Feather-provisioned phone number ID to call from. Must be a number registered in your organisation.
</ParamField>

<ParamField body="to_phone_number" type="string" required>
  The destination phone number in E.164 format, e.g. `+14155552671`.
</ParamField>

<ParamField body="agent_id" type="string">
  The agent to handle the call. Either `agent_id` or `team_id` must be provided.
</ParamField>

<ParamField body="team_id" type="string">
  Route the call to the team's default agent. Either `agent_id` or `team_id` must be provided.
</ParamField>

<ParamField body="context_variables" type="object">
  Key-value pairs injected into the agent's session context at call start. Use this to pass customer data (e.g. account ID, name) so the agent can personalise the call.

  ```json
  {
    "customer_name": "Alice Johnson",
    "account_id": "acc_9f2b1c3d",
    "reason_for_call": "renewal_reminder"
  }
  ```
</ParamField>

#### Response

<ResponseField name="room_name" type="string">
  The LiveKit room name for this call session. Useful for monitoring.
</ResponseField>

<ResponseField name="dispatch_id" type="string">
  Unique ID for this outbound dispatch job.
</ResponseField>

<ResponseField name="agent_name" type="string">
  Name of the agent assigned to handle the call.
</ResponseField>

<ResponseField name="session_id" type="string">
  The conversation session ID. Use this to retrieve the transcript and metadata after the call.
</ResponseField>

<Note>
  Outbound calls are dispatched asynchronously. The call may take a few seconds to connect depending on carrier routing. Use the `session_id` to poll for call completion and transcript availability.
</Note>

## Web Calls

Web calls use LiveKit to create a browser-based audio session. This is useful for:

- **Testing agents** without a physical phone
- **Embedding** voice assistants in web or mobile apps

### Start a Web Call

```
POST /v1/voice/web-call
```

<ParamField body="agent_id" type="string">
  The agent to connect to. Either `agent_id` or `team_id` is required.
</ParamField>

<ParamField body="team_id" type="string">
  Route to the team's default agent.
</ParamField>

<ParamField body="context_variables" type="object">
  Session context key-value pairs, same as outbound calls.
</ParamField>

#### Response

<ResponseField name="livekit_url" type="string">
  The LiveKit WebSocket URL to connect to.
</ResponseField>

<ResponseField name="access_token" type="string">
  Short-lived JWT access token for the LiveKit room. Pass this to `livekit-client.connect()`.
</ResponseField>

<ResponseField name="room_name" type="string">
  The LiveKit room name.
</ResponseField>

<ResponseField name="session_id" type="string">
  The conversation session ID for retrieving the transcript later.
</ResponseField>

#### Browser Connection Example

```javascript
import { Room } from "livekit-client";

const response = await fetch(
  "https://api-sandbox.featherhq.com/v1/voice/web-call",
  {
    method: "POST",
    headers: {
      "x-api-key": "YOUR_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      agent_id: "agent_support_01",
      context_variables: { customer_name: "Alice" },
    }),
  }
);

const { livekit_url, access_token } = await response.json();

const room = new Room();
await room.connect(livekit_url, access_token);
// Microphone is now live and connected to the AI agent
```

## Call Ingestion

Ingest a pre-recorded audio file (e.g. from a third-party telephony provider) for processing, transcription, and storage in Feather.

### Ingest a Call Recording

```
POST /v1/voice/calls
Content-Type: multipart/form-data
```

<ParamField body="audio" type="file" required>
  The call recording audio file. Supported formats: `audio/wav`, `audio/mpeg` (MP3), `audio/ogg`.
</ParamField>

<ParamField body="metadata" type="string" required>
  A JSON string containing call metadata.

  <Expandable title="metadata fields">
    <ParamField body="channel" type="string" required>
      Indicates the audio channel layout: `mono` or `stereo`. Use `stereo` when the caller and agent are on separate audio channels.
    </ParamField>
    <ParamField body="external_id" type="string">
      Your own identifier for this call, e.g. a telephony platform call SID. Stored for reference and deduplication.
    </ParamField>
    <ParamField body="language" type="string" default="en-US">
      BCP 47 language code for transcription.
    </ParamField>
  </Expandable>
</ParamField>

#### Response — `VoiceCallIngestResponse`

<ResponseField name="call_id" type="string">
  Feather's internal ID for the ingested call.
</ResponseField>

<ResponseField name="external_id" type="string">
  The external ID you provided, if any.
</ResponseField>

<ResponseField name="status" type="string">
  Processing status: `queued`, `transcribing`, or `complete`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of ingest.
</ResponseField>

## Retrieve a Call

```
GET /v1/voice/calls/{id}
```

Returns a processed call with its transcript and audio metadata.

#### Response — `VoiceCallGetResponse`

<ResponseField name="call" type="object">
  Call metadata.

  <Expandable title="call fields">
    <ResponseField name="id" type="string">
      Feather call ID.
    </ResponseField>
    <ResponseField name="session_id" type="string">
      Associated conversation session ID.
    </ResponseField>
    <ResponseField name="status" type="string">
      `in_progress`, `completed`, or `failed`.
    </ResponseField>
    <ResponseField name="started_at" type="string">
      Call start timestamp.
    </ResponseField>
    <ResponseField name="ended_at" type="string">
      Call end timestamp. `null` if in progress.
    </ResponseField>
    <ResponseField name="duration_seconds" type="integer">
      Total call duration in seconds.
    </ResponseField>
    <ResponseField name="from_number" type="string">
      The caller's phone number (may be redacted).
    </ResponseField>
    <ResponseField name="to_number" type="string">
      The dialled number.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="transcript" type="object[]">
  The call transcript, speaker-separated. PII may be scrubbed depending on org settings.

  <Expandable title="transcript entry fields">
    <ResponseField name="speaker" type="string">
      `agent` or `caller`.
    </ResponseField>
    <ResponseField name="text" type="string">
      Transcribed speech for this turn.
    </ResponseField>
    <ResponseField name="start_ms" type="integer">
      Start time offset from call start, in milliseconds.
    </ResponseField>
    <ResponseField name="end_ms" type="integer">
      End time offset, in milliseconds.
    </ResponseField>
    <ResponseField name="confidence" type="number">
      STT confidence score between `0.0` and `1.0`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="audio_url" type="string">
  Temporary signed URL to download the call recording. Valid for 15 minutes. Use `GET /v1/voice/recordings/{recording_id}/playback-url` for a fresh URL.
</ResponseField>

## Warm Transfer Detail

For calls where an agent consulted a supervisor before transferring:

```
GET /v1/voice/warm-transfer/{conversation_id}/transfer-detail
```

<ParamField path="conversation_id" type="string" required>
  The conversation session ID of the call.
</ParamField>

<ResponseField name="supervisor_phone" type="string">
  The phone number the call was transferred to.
</ResponseField>

<ResponseField name="failure_reason" type="string">
  If the transfer failed, the reason. `null` for successful transfers.
</ResponseField>

<ResponseField name="consult_recording" type="string">
  Signed URL to download the consultation recording (agent ↔ supervisor audio prior to full transfer).
</ResponseField>

<ResponseField name="consult_transcript" type="object[]">
  Transcript of the consultation phase. Same structure as the main `transcript` array.
</ResponseField>

## Recording Playback URL

Generate a fresh signed URL for a recording. The URL returned in `VoiceCallGetResponse.audio_url` expires after 15 minutes; use this endpoint to refresh it.

```
GET /v1/voice/recordings/{recording_id}/playback-url
```

<ParamField path="recording_id" type="string" required>
  The recording identifier.
</ParamField>

<ResponseField name="url" type="string">
  Fresh signed URL for the recording. Valid for 15 minutes.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 expiry of the signed URL.
</ResponseField>

## Examples

<CodeGroup>

```bash Place outbound call
curl -X POST https://api-sandbox.featherhq.com/v1/voice/outbound \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "from_phone_number_id": "pn_01hx3rp2v9abcdef",
    "to_phone_number": "+14155552671",
    "agent_id": "agent_support_01",
    "context_variables": {
      "customer_name": "Alice Johnson",
      "account_id": "acc_9f2b1c3d",
      "reason_for_call": "renewal_reminder"
    }
  }'
```

```bash Start web call
curl -X POST https://api-sandbox.featherhq.com/v1/voice/web-call \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "agent_support_01",
    "context_variables": {
      "customer_name": "Bob"
    }
  }'
```

```bash Ingest recording
curl -X POST https://api-sandbox.featherhq.com/v1/voice/calls \
  -H "x-api-key: YOUR_API_KEY" \
  -F "audio=@/path/to/call_recording.wav" \
  -F 'metadata={"channel":"stereo","external_id":"twilio_CA1234567890","language":"en-US"}'
```

```bash Get call with transcript
curl https://api-sandbox.featherhq.com/v1/voice/calls/call_4b2a9c1d \
  -H "x-api-key: YOUR_API_KEY"
```

```bash Refresh playback URL
curl https://api-sandbox.featherhq.com/v1/voice/recordings/rec_7e3f1a2b/playback-url \
  -H "x-api-key: YOUR_API_KEY"
```

</CodeGroup>

```json
{
  "room_name": "room_feather_7b3a1c2d",
  "dispatch_id": "disp_4f2e9a1b",
  "agent_name": "Support Agent",
```

<Warning>
  Transcripts may contain redacted PII depending on your organisation's data handling settings. Redacted segments appear as `[REDACTED]` in the `text` field. Configure PII scrubbing under Organisation Settings.
</Warning>

<Tip>
  Use `context_variables` to pass the full CRM customer record at call start. This allows your agent to greet the caller by name and reference their account without asking for identifying information.
</Tip>
