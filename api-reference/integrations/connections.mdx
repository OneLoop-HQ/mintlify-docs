## Overview

Integrations let you connect third-party services to Feather so agents can use their tools and knowledge base connectors can pull from their content. The lifecycle is:

1. **Browse** available integrations
2. **Create** a connection and get an OAuth URL
3. **Finalize** the connection after the user completes the OAuth flow
4. **Use** the synced tools in your agents

**Base URL:** `https://api-sandbox.featherhq.com`

## Available Integrations

### List Available Integrations

```
GET /v1/integrations
```

Returns all integrations available in the Feather marketplace.

#### Response — `IntegrationSummary[]`

<ResponseField name="key" type="string">
  Unique identifier for the integration, e.g. `notion`, `google_drive`, `salesforce`.
</ResponseField>

<ResponseField name="display_name" type="string">
  Human-readable name shown in UIs, e.g. `"Notion"`.
</ResponseField>

<ResponseField name="description" type="string">
  Short description of what the integration provides.
</ResponseField>

<ResponseField name="category" type="string">
  Integration category, e.g. `productivity`, `crm`, `storage`, `communication`.
</ResponseField>

<ResponseField name="logo_url" type="string">
  URL of the integration's logo image.
</ResponseField>

## Connections

A **connection** is an active authorised link between your organisation and a specific integration.

### List Connections

```
GET /v1/integrations/connections
```

Returns all integration connections for the organisation, including their current status and synced tools.

---

### Create a Connection (Start OAuth)

```
POST /v1/integrations/connections
```

Initiates the OAuth flow. Returns a session token and URL to open in a browser popup.

<ParamField body="integration_key" type="string" required>
  The key of the integration to connect, e.g. `notion` or `salesforce`.
</ParamField>

<ParamField body="nickname" type="string">
  A friendly name for this connection, e.g. `"Marketing Notion Workspace"`. Useful when you have multiple connections to the same provider.
</ParamField>

#### Response

<ResponseField name="connection_id" type="string">
  The new connection's ID. Use this to finalize after the OAuth flow completes.
</ResponseField>

<ResponseField name="session_token" type="string">
  Short-lived token used to authenticate the OAuth popup session.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 expiry of the `session_token`. The OAuth flow must be completed before this time.
</ResponseField>

<ResponseField name="vendor" type="string">
  The integration vendor name, for display purposes.
</ResponseField>

---

### Finalize a Connection

```
POST /v1/integrations/connections/{id}/finalize
```

Call this after the user completes the OAuth popup. Feather exchanges the auth code for access tokens and begins syncing tools.

<ParamField path="id" type="string" required>
  The connection ID returned from the create step.
</ParamField>

<Note>
  After finalization, tool sync runs asynchronously. Check `tools_status` on the connection to know when tools are ready to use.
</Note>

---

### Get a Connection

```
GET /v1/integrations/connections/{id}
```

#### Response — `ConnectionResponse`

<ResponseField name="id" type="string">
  Connection identifier.
</ResponseField>

<ResponseField name="integration_key" type="string">
  The integration this connection belongs to.
</ResponseField>

<ResponseField name="nickname_slug" type="string">
  URL-safe slug generated from the nickname.
</ResponseField>

<ResponseField name="nickname" type="string">
  Human-readable connection name.
</ResponseField>

<ResponseField name="status" type="string">
  Connection lifecycle status:

  | Value | Meaning |
  |---|---|
  | `active` | Connection is authorised and working |
  | `pending_oauth` | Awaiting OAuth flow completion |
</ResponseField>

<ResponseField name="tools_status" type="string">
  Status of the tool sync from the connected service:

  | Value | Meaning |
  |---|---|
  | `syncing` | Tool list is being fetched from the provider |
  | `synced` | Tools are ready to use |
  | `failed` | Tool sync failed; retry with `refresh-tools` |
</ResponseField>

<ResponseField name="tools" type="ConnectionToolResponse[]">
  List of tools available through this connection.

  <Expandable title="ConnectionToolResponse fields">
    <ResponseField name="id" type="string">
      Tool identifier within this connection.
    </ResponseField>
    <ResponseField name="name" type="string">
      Tool display name.
    </ResponseField>
    <ResponseField name="slug" type="string">
      URL-safe slug for the tool.
    </ResponseField>
    <ResponseField name="description" type="string">
      What the tool does.
    </ResponseField>
    <ResponseField name="enabled" type="boolean">
      Whether the tool is enabled for use by agents.
    </ResponseField>
  </Expandable>
</ResponseField>

### Update a Connection

```
PATCH /v1/integrations/connections/{id}
```

<ParamField body="nickname" type="string">
  Rename the connection.
</ParamField>

---

### Delete a Connection

```
DELETE /v1/integrations/connections/{id}
```

Disconnects the integration and revokes Feather's access tokens. All KB sources using this connection will stop syncing.

<Warning>
  Deleting a connection does not remove documents that were already indexed from it. Remove those separately from the KB documents API if needed.
</Warning>

---

### Refresh Tools

```
POST /v1/integrations/connections/{id}/refresh-tools
```

Re-fetches the tool list from the connected provider. Use this when the integration adds new actions or when `tools_status` is `failed`.

## Email Integration

Connect Feather to a platform-level email integration, separate from standard OAuth connections.

```
POST /v1/integrations/connections/email
```

<ParamField body="provider" type="string" required>
  Email provider, e.g. `gmail` or `outlook`.
</ParamField>

<ParamField body="integration_connection_id" type="string">
  Existing OAuth connection ID to use for email access.
</ParamField>

## Webhook Events

Inspect recent webhook events delivered to your integration connections for debugging.

```
GET /v1/integrations/webhook-events
```

<ResponseField name="items" type="object[]">
  List of recent webhook events, newest first.

  <Expandable title="webhook event fields">
    <ResponseField name="id" type="string">
      Event ID.
    </ResponseField>
    <ResponseField name="connection_id" type="string">
      The connection that received this event.
    </ResponseField>
    <ResponseField name="event_type" type="string">
      Provider-specific event type, e.g. `page.updated`, `file.created`.
    </ResponseField>
    <ResponseField name="received_at" type="string">
      ISO 8601 timestamp when Feather received the event.
    </ResponseField>
    <ResponseField name="processed" type="boolean">
      Whether Feather successfully processed the event.
    </ResponseField>
    <ResponseField name="error_message" type="string">
      Processing error, if any.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash List available integrations
curl https://api-sandbox.featherhq.com/v1/integrations \
  -H "x-api-key: YOUR_API_KEY"
```

```bash Create a connection
curl -X POST https://api-sandbox.featherhq.com/v1/integrations/connections \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "integration_key": "notion",
    "nickname": "Marketing Workspace"
  }'
```

```bash Finalize after OAuth
curl -X POST https://api-sandbox.featherhq.com/v1/integrations/connections/conn_9f1a2b3c/finalize \
  -H "x-api-key: YOUR_API_KEY"
```

```bash Get connection with tools
curl https://api-sandbox.featherhq.com/v1/integrations/connections/conn_9f1a2b3c \
  -H "x-api-key: YOUR_API_KEY"
```

```bash Refresh tools
curl -X POST https://api-sandbox.featherhq.com/v1/integrations/connections/conn_9f1a2b3c/refresh-tools \
  -H "x-api-key: YOUR_API_KEY"
```

</CodeGroup>
