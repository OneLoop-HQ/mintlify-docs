## Overview

You can invoke any tool directly — without running an agent conversation — using the execute and test endpoints. This is useful for:

- **Integration testing** during development
- **Debugging** unexpected tool responses in production
- **Previewing field mappings** before promoting a revision to active

**Base URL:** `https://api-sandbox.featherhq.com`

---

## Execute a Tool

Executes the tool's **active revision** with the inputs you provide.

```
POST /v1/tools/{id}/execute
```

<ParamField path="id" type="string" required>
  The tool ID to execute.
</ParamField>

<ParamField body="input" type="object" required>
  Key-value map of variable values matching the tool's declared `variables`. Required variables must be present.

  ```json
  {
    "order_id": "ORD-98210",
    "region": "us-east"
  }
  ```
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary metadata attached to this execution for logging or tracing purposes. Not forwarded to the external API.
</ParamField>

### Response — `ToolExecuteResponse`

<ResponseField name="status_code" type="integer">
  The HTTP status code returned by the external API.
</ResponseField>

<ResponseField name="headers" type="object">
  Response headers from the external API.
</ResponseField>

<ResponseField name="body" type="object">
  Parsed response body from the external API. If the response is not valid JSON, this field contains `{ "raw": "<string>" }`.
</ResponseField>

<ResponseField name="success" type="boolean">
  `true` if the request returned a 2xx status code and did not time out.
</ResponseField>

<ResponseField name="error" type="string">
  Present when `success` is `false`. Human-readable error description, e.g. `"Connection timed out after 8000ms"`.
</ResponseField>

<ResponseField name="execution_time_ms" type="integer">
  Total round-trip time in milliseconds, including any retry attempts.
</ResponseField>

<ResponseField name="request_url" type="string">
  The fully-resolved URL that was called, including substituted path variables and query parameters. Useful for debugging.
</ResponseField>

<ResponseField name="request_method" type="string">
  The HTTP method used for the request.
</ResponseField>

<Note>
  The `request_url` field substitutes all `{{variable}}` placeholders with the values you provided in `input`. Sensitive headers are redacted and not reflected in this response.
</Note>

## Test a Specific Revision

Tests a **specific revision** — useful for validating configuration before promoting it to active. Optionally provide a `response_field_map` to preview how field mappings transform the response.

```
POST /v1/tools/{id}/revisions/{revision_id}/test
```

<ParamField path="id" type="string" required>
  The tool ID.
</ParamField>

<ParamField path="revision_id" type="string" required>
  The specific revision to test.
</ParamField>

<ParamField body="input" type="object" required>
  Variable values for the test call.
</ParamField>

<ParamField body="response_field_map" type="object[]">
  Optional field mapping rules to apply to the response. If omitted, the revision's stored `response_schema.field_mappings` are used.

  ```json
  [
    { "key": "status", "path": "$.data.state" },
    { "key": "eta", "path": "$.data.delivery.estimated_date" }
  ]
  ```
</ParamField>

### Response — `ToolRevisionTestResponse`

<ResponseField name="status_code" type="integer">
  HTTP status code from the external API.
</ResponseField>

<ResponseField name="body" type="object">
  The raw, unprocessed response body.
</ResponseField>

<ResponseField name="narrowed" type="object">
  The response body after applying the field mappings. Keys are the `key` values from `response_field_map`; values are the extracted data. `null` if no field mappings were applied.
</ResponseField>

<ResponseField name="success" type="boolean">
  `true` if the request succeeded.
</ResponseField>

<ResponseField name="error" type="string">
  Error description if `success` is `false`.
</ResponseField>

<ResponseField name="execution_time_ms" type="integer">
  Round-trip duration in milliseconds.
</ResponseField>

<ResponseField name="request_url" type="string">
  The fully-resolved request URL.
</ResponseField>

<ResponseField name="request_method" type="string">
  HTTP method used.
</ResponseField>

## Full Walkthrough Example

This example creates a tool that calls `https://httpbin.org/get` with a query parameter, executes it directly, and previews a field mapping.

### Step 1 — Create the tool

<CodeGroup>

```bash Create tool
curl -X POST https://api-sandbox.featherhq.com/v1/tools \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "httpbin_echo",
    "description": "Echoes a message via httpbin GET for testing purposes.",
    "tool_type": "api_call",
    "configuration": {
      "url": "https://httpbin.org/get",
      "method": "GET",
      "query_params": [
        { "key": "message", "value": "{{message}}" }
      ],
      "timeout": 5000,
      "variables": [
        {
          "name": "message",
          "type": "string",
          "description": "The message to echo back.",
          "required": true
        }
      ]
    }
  }'
```

</CodeGroup>

**Response:**

```json
{
  "id": "tool_echo_abc123",
  "name": "httpbin_echo",
  "tool_type": "api_call",
  "enabled": true,
  "active_revision": {
    "id": "rev_echo_001",
    "version": 1,
    "created_at": "2024-06-01T10:00:00Z"
  },
  "created_at": "2024-06-01T10:00:00Z"
}
```

### Step 2 — Execute the tool directly

<CodeGroup>

```bash Execute tool
curl -X POST https://api-sandbox.featherhq.com/v1/tools/tool_echo_abc123/execute \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "input": {
      "message": "Hello from Feather!"
    },
    "metadata": {
      "test_run": true
    }
  }'
```

</CodeGroup>

**Response:**

```json
{
  "status_code": 200,
  "headers": {
    "Content-Type": "application/json",
    "Server": "gunicorn/19.9.0"
  },
  "body": {
    "args": {
      "message": "Hello from Feather!"
    },
    "headers": {
      "Host": "httpbin.org",
      "User-Agent": "FeatherCX/1.0"
    },
    "origin": "34.120.50.100",
    "url": "https://httpbin.org/get?message=Hello+from+Feather!"
  },
  "success": true,
  "error": null,
  "execution_time_ms": 312,
  "request_url": "https://httpbin.org/get?message=Hello+from+Feather!",
  "request_method": "GET"
}
```

### Step 3 — Test a revision with field mappings

<CodeGroup>

```bash Test revision with field map
curl -X POST https://api-sandbox.featherhq.com/v1/tools/tool_echo_abc123/revisions/rev_echo_001/test \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "input": {
      "message": "Field mapping test"
    },
    "response_field_map": [
      { "key": "echoed_message", "path": "$.args.message" },
      { "key": "origin_ip", "path": "$.origin" }
    ]
  }'
```

</CodeGroup>

**Response:**

```json
{
  "status_code": 200,
  "body": {
    "args": { "message": "Field mapping test" },
    "origin": "34.120.50.100",
    "url": "https://httpbin.org/get?message=Field+mapping+test"
  },
  "narrowed": {
    "echoed_message": "Field mapping test",
    "origin_ip": "34.120.50.100"
  },
  "success": true,
  "error": null,
  "execution_time_ms": 287,
  "request_url": "https://httpbin.org/get?message=Field+mapping+test",
  "request_method": "GET"
}
```

<Tip>
  Use the `narrowed` field to verify that your JSONPath expressions correctly extract the fields you want before attaching the tool to a live agent.
</Tip>

<Warning>
  Direct tool execution counts against your API rate limits. In high-volume testing scenarios, use the revision test endpoint with a sandbox or mock URL to avoid exhausting external API quotas.
</Warning>
