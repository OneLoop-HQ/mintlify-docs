## Overview

Feather notifications push platform events to Slack channels. You configure destinations (where to send), routes (which event types go where), and templates (how each event looks). A settings object controls global defaults like timezone and quiet hours.

---

## Destinations

A destination is a Slack channel that Feather has permission to post to. Feather verifies the connection when you create a destination.

### Discover Available Channels

```
GET /v1/notifications/destinations/discover?channel_kind=slack
```

Returns a list of Slack channels the connected bot has permission to post to. Use this to let users select a channel before creating a destination.

<ParamField query="channel_kind" type="string" required>
  Currently only `slack` is supported.
</ParamField>

### Create a Destination

```
POST /v1/notifications/destinations
```

#### Request Body

<ParamField body="channel_kind" type="string" required>
  `slack`.
</ParamField>

<ParamField body="target_id" type="string" required>
  The Slack channel ID (e.g. `C08ABC1234`). Use the discovery endpoint to find valid IDs.
</ParamField>

<ParamField body="target_label" type="string" required>
  Human-readable label for this destination, e.g. `#cx-alerts`.
</ParamField>

<ParamField body="connection_id" type="string (uuid)" required>
  The Feather integration connection ID for your Slack workspace.
</ParamField>

#### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/notifications/destinations \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "channel_kind": "slack",
    "target_id": "C08ABC1234",
    "target_label": "#cx-alerts",
    "connection_id": "conn_aabb1122-0000-0000-0000-000000000010"
  }'
```

```json Response
{
  "id": "dest_ccdd3344-0000-0000-0000-000000000020",
  "channel_kind": "slack",
  "target_id": "C08ABC1234",
  "target_label": "#cx-alerts",
  "verified": true,
  "created_at": "2024-06-01T09:00:00Z"
}
```
</CodeGroup>

### List Destinations

```
GET /v1/notifications/destinations
```

### Delete a Destination

```
DELETE /v1/notifications/destinations/{id}
```

## Routes

A route connects an event type to a destination. One event type can be routed to multiple destinations, and one destination can receive multiple event types.

### Create or Update a Route

```
PUT /v1/notifications/routes
```

Creates a new route or updates an existing one if a route for this `event_type` + `destination_id` combination already exists.

#### Request Body

<ParamField body="event_type" type="string" required>
  The event type to route (e.g. `policy.violation.detected`). See the [Webhooks event catalog](/api-reference/webhooks#get-the-event-catalog) for all types.
</ParamField>

<ParamField body="destination_id" type="string (uuid)" required>
  The destination to send matching events to.
</ParamField>

<ParamField body="enabled" type="boolean" required>
  Whether this route is active.
</ParamField>

#### Example

<CodeGroup>
```bash cURL
curl -X PUT https://api-sandbox.featherhq.com/v1/notifications/routes \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "policy.violation.detected",
    "destination_id": "dest_ccdd3344-0000-0000-0000-000000000020",
    "enabled": true
  }'
```
</CodeGroup>

### List Routes

```
GET /v1/notifications/routes
```

### Delete a Route

```
DELETE /v1/notifications/routes/{id}
```

## Notification Types

Notification types are the event catalog with per-organization configuration layered on top. Each type can be enabled/disabled, have its severity overridden, and have rate-limiting applied.

### List Notification Types

```
GET /v1/notifications/types
```

Returns the full catalog with your organization's configuration for each type.

### Configure a Notification Type

```
PUT /v1/notifications/types/{event_type}/config
```

#### Request Body

<ParamField body="enabled" type="boolean">
  Enable or disable this notification type.
</ParamField>

<ParamField body="severity_override" type="string">
  Override the default severity: `info`, `low`, `medium`, `high`, `critical`.
</ParamField>

<ParamField body="threshold_config" type="object">
  For count-based notifications, configure the threshold that must be exceeded before a notification fires.
</ParamField>

<ParamField body="delivery_mode" type="string">
  `immediate` — Send as soon as the event occurs.
</ParamField>

<ParamField body="rate_limit_per_hour" type="integer">
  Maximum notifications of this type to send per hour. Excess events are dropped, not queued.
</ParamField>

### Enable Recommended Types

```
POST /v1/notifications/types/enable-recommended
```

One-click enables the recommended starter set of notification types with sensible defaults. Ideal for getting started quickly.

## Templates

Templates control how each notification type renders in Slack. Each template has named slots.

### Get a Template

```
GET /v1/notifications/types/{event_type}/template?channel_kind=slack
```

#### Query Parameters

<ParamField query="channel_kind" type="string" required>
  `slack`.
</ParamField>

### Update a Template

```
PUT /v1/notifications/types/{event_type}/template?channel_kind=slack
```

#### Request Body

<ParamField body="title" type="string">
  Notification title. Supports template variables (e.g. `{{agent_name}}`).
</ParamField>

<ParamField body="body" type="string">
  Main notification body text. Supports Slack markdown and template variables.
</ParamField>

<ParamField body="fields" type="array of objects">
  Key-value fields rendered as a Slack attachment block. Each entry: `label` and `value` (both support template variables).
</ParamField>

<ParamField body="link_text" type="string">
  Label for the call-to-action link button.
</ParamField>

<ParamField body="link_url" type="string">
  URL for the call-to-action link button.
</ParamField>

### Delete a Template

```
DELETE /v1/notifications/types/{event_type}/template
```

Resets the template to the Feather default for this event type.

### Validate a Template

```
POST /v1/notifications/types/{event_type}/template/validate
```

Validates template syntax and variable references without saving. Returns a list of validation errors if any exist.

### Preview a Template

```
POST /v1/notifications/types/{event_type}/template/preview
```

Renders the template with sample data and returns a preview payload you can inspect before saving.

## Settings

```
GET /v1/notifications/settings
PUT /v1/notifications/settings
```

Organization-wide notification preferences.

### Request Body (PUT)

<ParamField body="default_destination_id" type="string (uuid)">
  Destination used when an event type has no explicit route configured.
</ParamField>

<ParamField body="timezone" type="string">
  IANA timezone for quiet hours evaluation (e.g. `America/New_York`).
</ParamField>

<ParamField body="quiet_hours" type="object">
  Time window during which non-critical notifications are suppressed:
  - `start` (string) — Start time in `HH:MM` format.
  - `end` (string) — End time in `HH:MM` format.
  - `days` (array of strings) — Days the quiet hours apply, e.g. `["saturday", "sunday"]`.
</ParamField>

## Delivery Log

```
GET /v1/notifications/delivery-log
```

Returns a log of notification delivery attempts — successes, failures, and rate-limited drops.

## Test Send

```
POST /v1/notifications/types/{event_type}/test-send
```

Sends a test notification for the given event type to all active routes. Use this to verify your destination and template configuration before relying on live events.

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/notifications/types/policy.violation.detected/test-send \
  -H "x-api-key: <key>"
```

```json Response
{
  "event_type": "policy.violation.detected",
  "destinations_notified": ["dest_ccdd3344-0000-0000-0000-000000000020"],
  "status": "sent"
}
```
</CodeGroup>

<Tip>
  After creating your first destination and route, run a test send to confirm the Slack bot can post to your chosen channel before enabling live event routing.
</Tip>
