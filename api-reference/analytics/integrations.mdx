## Overview

Integration analytics give you visibility into two dimensions of your integration layer:

1. **Tool Calls** — How your integration tools are performing (volume, latency, errors).
2. **Connection Health** — How reliably your integration connections are staying connected.

All endpoints live under `/v1/analytics/integrations`. Unless otherwise noted, `start_date` and `end_date` (`YYYY-MM-DD`) are required.

---

## Tool Call Analytics

### Summary

```
GET /v1/analytics/integrations/summary
```

Returns headline tool-call KPIs for all integrations in the window: `total_calls`, `success_rate_pct`, `avg_latency_ms`, `total_errors`.

### Usage Timeseries

```
GET /v1/analytics/integrations/usage-timeseries
```

Returns integration tool call volume over time.

#### Query Parameters

<ParamField query="granularity" type="string">
  `hour`, `day`, or `week`. Default: `day`.
</ParamField>

<ParamField query="integration_id" type="string">
  Filter to a specific integration.
</ParamField>

### Success Rate Timeseries

```
GET /v1/analytics/integrations/success-rate-timeseries
```

Returns tool call success rate over time. Each row: `time_bucket`, `success_rate_pct`, `total_calls`, `failed_calls`.

### Latency Timeseries

```
GET /v1/analytics/integrations/latency-timeseries
```

Returns p50, p90, and p95 latency for integration tool calls over time.

### By Integration

```
GET /v1/analytics/integrations/by-integration
```

Returns per-integration aggregates: `integration_id`, `integration_name`, `total_calls`, `success_rate_pct`, `avg_latency_ms`, `error_count`.

### By Mode

```
GET /v1/analytics/integrations/by-mode
```

Returns tool call counts and success rates broken down by connection mode (e.g. `oauth`, `api_key`, `service_account`).

### Error Breakdown

```
GET /v1/analytics/integrations/error-breakdown
```

Returns error counts grouped by error class: `timeout`, `auth_failure`, `rate_limit`, `invalid_response`, `unknown`.

### Tool Call Dashboard

```
GET /v1/analytics/integrations/dashboard
```

Returns all tool-call analytics in a single envelope.

#### Response: DashboardEnvelope

<ResponseField name="summary" type="array of objects">
  Headline KPIs per integration.
</ResponseField>

<ResponseField name="usage" type="array of objects">
  Usage timeseries data.
</ResponseField>

<ResponseField name="success_rate" type="array of objects">
  Success rate timeseries.
</ResponseField>

<ResponseField name="latency" type="array of objects">
  Latency timeseries.
</ResponseField>

<ResponseField name="by_integration" type="array of objects">
  Per-integration aggregates.
</ResponseField>

<ResponseField name="by_mode" type="array of objects">
  Mode breakdown.
</ResponseField>

<ResponseField name="error_breakdown" type="array of objects">
  Error class breakdown.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/analytics/integrations/dashboard?start_date=2024-06-01&end_date=2024-06-30" \
  -H "x-api-key: <key>"
```

```json Response
{
  "summary": [
    {
      "integration_id": "intg_11aa22bb-0000-0000-0000-000000000040",
      "integration_name": "Salesforce CRM",
      "total_calls": 12400,
      "success_rate_pct": 98.1,
      "avg_latency_ms": 340,
      "total_errors": 236
    }
  ],
  "error_breakdown": [
    { "error_class": "timeout", "count": 180 },
    { "error_class": "auth_failure", "count": 56 }
  ]
}
```
</CodeGroup>

## Connection Health Analytics

### Connection Summary

```
GET /v1/analytics/integrations/connections/summary
```

Returns aggregate connection lifecycle metrics across all integrations.

#### Response

<ResponseField name="events" type="integer">
  Total connection lifecycle events in the window.
</ResponseField>

<ResponseField name="successes" type="integer">
  Successful connection events (connects, reconnects).
</ResponseField>

<ResponseField name="failures" type="integer">
  Failed connection events.
</ResponseField>

<ResponseField name="success_rate_pct" type="number">
  Percentage of connection events that succeeded.
</ResponseField>

<ResponseField name="avg_latency_ms" type="number">
  Average connection establishment latency.
</ResponseField>

<ResponseField name="p95_latency_ms" type="number">
  95th percentile connection latency.
</ResponseField>

### Connections By Integration

```
GET /v1/analytics/integrations/connections/by-integration
```

Returns the connection health summary broken down per integration.

### Connection Events Timeseries

```
GET /v1/analytics/integrations/connections/events-timeseries
```

Returns connection event counts (connect, disconnect, reconnect, error) over time.

#### Query Parameters

<ParamField query="granularity" type="string">
  `hour`, `day`, or `week`. Default: `day`.
</ParamField>

<ParamField query="event_type" type="string">
  Filter to a specific event type: `connect`, `disconnect`, `reconnect`, `error`.
</ParamField>

### Connection Success Rate Timeseries

```
GET /v1/analytics/integrations/connections/success-rate-timeseries
```

Returns connection success rate over time. Each row: `time_bucket`, `success_rate_pct`, `total_events`.

### Connection Failure Breakdown

```
GET /v1/analytics/integrations/connections/failure-breakdown
```

Returns connection failures grouped by failure reason (e.g. `token_expired`, `network_error`, `permission_denied`, `rate_limited`).

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/analytics/integrations/connections/failure-breakdown?start_date=2024-06-01&end_date=2024-06-30" \
  -H "x-api-key: <key>"
```

```json Response
{
  "data": [
    { "reason": "token_expired", "count": 42 },
    { "reason": "network_error", "count": 18 },
    { "reason": "permission_denied", "count": 7 }
  ]
}
```
</CodeGroup>
