## Overview

Tool analytics are organized into three groups — **Usage**, **Performance**, and **Reliability** — plus a single **Dashboard** endpoint that returns all key metrics in one request. All endpoints live under `/v1/analytics/tools`.

---

## Usage Endpoints

### By Type

```
GET /v1/analytics/tools/usage/by-type
```

Returns invocation counts grouped by tool type (e.g. `api`, `integration`, `built_in`).

### By Tool

```
GET /v1/analytics/tools/usage/by-tool
```

Returns per-tool invocation counts. Useful for spotting the most-called tools.

### By Agent

```
GET /v1/analytics/tools/usage/by-agent
```

Returns tool invocation counts grouped by the agent that called them.

### Unique Conversations

```
GET /v1/analytics/tools/usage/unique-conversations
```

Returns the number of unique conversations in which each tool was invoked at least once.

### Usage Timeseries

```
GET /v1/analytics/tools/usage/timeseries
```

Returns tool invocation volume over time.

#### Query Parameters

<ParamField query="start_date" type="string" required>
  Start date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="end_date" type="string" required>
  End date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="granularity" type="string">
  Time bucket: `hour`, `day`, `week`. Default: `day`.
</ParamField>

<ParamField query="tool_id" type="string">
  Filter to a specific tool.
</ParamField>

## Performance Endpoints

### Latency Timeseries

```
GET /v1/analytics/tools/performance/timeseries
```

Returns tool call latency percentiles over time.

### Latency By Tool

```
GET /v1/analytics/tools/performance/by-tool
```

Returns average, p50, p90, p95, and p99 latency per tool.

### Latency Distribution

```
GET /v1/analytics/tools/performance/distribution
```

Returns a histogram of tool call durations in millisecond buckets.

### Latency By Provider

```
GET /v1/analytics/tools/performance/by-provider
```

Returns latency aggregated by integration provider (e.g. Salesforce, Zendesk, custom API).

### Retry Analysis

```
GET /v1/analytics/tools/performance/retry-analysis
```

Returns retry statistics per tool — retry rate, mean retries before success, and retry-induced latency overhead.

---

## Reliability Endpoints

### Success Rate Timeseries

```
GET /v1/analytics/tools/reliability/success-rate-timeseries
```

Returns tool call success rates over time.

### Error Breakdown

```
GET /v1/analytics/tools/reliability/error-breakdown
```

Returns error counts grouped by error class (e.g. `timeout`, `auth_failure`, `invalid_response`).

### Error Breakdown By Status

```
GET /v1/analytics/tools/reliability/error-breakdown-by-status
```

Returns error counts grouped by HTTP status code.

### Tool Ranking

```
GET /v1/analytics/tools/reliability/ranking
```

Returns tools ranked by error rate descending — surfaces the most problematic tools first.

### Error Timeseries

```
GET /v1/analytics/tools/reliability/error-timeseries
```

Returns raw error counts over time.

### Error Trend

```
GET /v1/analytics/tools/reliability/error-trend
```

Returns a week-over-week or period-over-period error trend per tool to surface improving or degrading reliability.

### Timeout Analysis

```
GET /v1/analytics/tools/reliability/timeout-analysis
```

Returns timeout counts and timeout rate per tool, grouped by configured timeout threshold.

## Dashboard (All-in-One)

```
GET /v1/analytics/tools/dashboard
```

Returns a single envelope containing all key tool analytics in one API call. Ideal for populating monitoring dashboards.

### Query Parameters

<ParamField query="start_date" type="string" required>
  Start date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="end_date" type="string" required>
  End date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="tool_type" type="string">
  Filter all metrics to a specific tool type.
</ParamField>

<ParamField query="revision_id" type="string">
  Filter to a specific agent revision.
</ParamField>

<ParamField query="top_tools_limit" type="integer">
  Max tools to return in the `top_tools` section. Default: `10`.
</ParamField>

<ParamField query="health_matrix_limit" type="integer">
  Max tools to return in the `health_matrix` section. Default: `20`.
</ParamField>

<ParamField query="recent_errors_limit" type="integer">
  Max recent error events to include. Default: `25`.
</ParamField>

<ParamField query="min_invocations" type="integer">
  Exclude tools with fewer than this many invocations from rankings.
</ParamField>

<ParamField query="latency_ceiling_ms" type="integer">
  Flag tools with p95 latency above this threshold in the health matrix.
</ParamField>

### Response

<ResponseField name="summary" type="object">
  Headline KPIs: `total_invocations`, `overall_success_rate_pct`, `avg_latency_ms`, `total_errors`.
</ResponseField>

<ResponseField name="top_tools" type="array of objects">
  Most-invoked tools, each with `tool_id`, `tool_name`, `invocation_count`.
</ResponseField>

<ResponseField name="health_matrix" type="array of DashboardHealthMatrixRow">
  Per-tool health overview. See [DashboardHealthMatrixRow](#dashboardhealthmatrixrow) below.
</ResponseField>

<ResponseField name="recent_errors" type="array of objects">
  Most recent tool error events with `tool_id`, `error_class`, `error_message`, `occurred_at`.
</ResponseField>

### DashboardHealthMatrixRow

<ResponseField name="tool_id" type="string">
  Unique tool identifier.
</ResponseField>

<ResponseField name="tool_name" type="string">
  Human-readable tool name.
</ResponseField>

<ResponseField name="invocation_count" type="integer">
  Total invocations in the window.
</ResponseField>

<ResponseField name="success_rate" type="number">
  Fraction of successful invocations (0.0–1.0).
</ResponseField>

<ResponseField name="p95_latency_ms" type="number">
  95th percentile latency in milliseconds.
</ResponseField>

<ResponseField name="health_score" type="number">
  Composite health score (0–100) combining success rate, latency, and error trend.
</ResponseField>
