## Overview

Session analytics endpoints let you monitor the health and volume of conversations over time. All endpoints require `start_date` and `end_date` query parameters in `YYYY-MM-DD` format.

**Base path:** `/v1/analytics/sessions`

---

## Common Query Parameters

All session analytics endpoints accept the following base parameters:

<ParamField query="start_date" type="string" required>
  Start of the reporting window. Format: `YYYY-MM-DD` (e.g. `2024-06-01`).
</ParamField>

<ParamField query="end_date" type="string" required>
  End of the reporting window, inclusive. Format: `YYYY-MM-DD`.
</ParamField>

---

## 1. Volume Timeseries

```
GET /v1/analytics/sessions/volume-timeseries
```

Returns session creation counts broken out by time bucket, channel, and status.

### Additional Query Parameters

<ParamField query="granularity" type="string">
  Time bucket size: `hour`, `day`, `week`, or `month`. Default: `day`.
</ParamField>

<ParamField query="channel" type="string">
  Filter to a specific channel: `sms`, `email`, `voice`, `chat`, etc.
</ParamField>

<ParamField query="status" type="string">
  Filter by session status.
</ParamField>

<ParamField query="priority" type="string">
  Filter by session priority.
</ParamField>

### Response: VolumeTimeseriesRow

<ResponseField name="time_bucket" type="string (ISO 8601)">
  The start of the time bucket.
</ResponseField>

<ResponseField name="channel" type="string">
  The communication channel.
</ResponseField>

<ResponseField name="sessions_created" type="integer">
  Number of sessions created in this bucket.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/analytics/sessions/volume-timeseries?start_date=2024-06-01&end_date=2024-06-07&granularity=day&channel=sms" \
  -H "x-api-key: <key>"
```

```json Response
{
  "data": [
    { "time_bucket": "2024-06-01T00:00:00Z", "channel": "sms", "sessions_created": 142 },
    { "time_bucket": "2024-06-02T00:00:00Z", "channel": "sms", "sessions_created": 168 },
    { "time_bucket": "2024-06-03T00:00:00Z", "channel": "sms", "sessions_created": 131 }
  ]
}
```
</CodeGroup>

## 3. Resolution Time

```
GET /v1/analytics/sessions/resolution-time
```

Returns percentile latency stats for how long sessions take to resolve from creation.

### Additional Query Parameters

<ParamField query="channel" type="string">
  Filter to a specific channel.
</ParamField>

<ParamField query="priority" type="string">
  Filter by priority.
</ParamField>

### Response: ResolutionTimeRow

<ResponseField name="avg_seconds" type="number">Average resolution time in seconds.</ResponseField>
<ResponseField name="p50_seconds" type="number">50th percentile (median) resolution time.</ResponseField>
<ResponseField name="p90_seconds" type="number">90th percentile resolution time.</ResponseField>
<ResponseField name="p95_seconds" type="number">95th percentile resolution time.</ResponseField>
<ResponseField name="p99_seconds" type="number">99th percentile resolution time.</ResponseField>

---

## 4. First Response Time

```
GET /v1/analytics/sessions/first-response-time
```

Returns time-to-first-response percentiles — the time between session creation and the agent's first reply.

### Additional Query Parameters

<ParamField query="channel" type="string">
  Filter to a specific channel.
</ParamField>

### Response: FirstResponseTimeRow

Same percentile fields as [ResolutionTimeRow](#response-resolutiontimerow): `avg_seconds`, `p50_seconds`, `p90_seconds`, `p95_seconds`, `p99_seconds`.

## 8. Turn Depth

```
GET /v1/analytics/sessions/turn-depth
```

Returns aggregate turn-depth statistics — how many back-and-forth messages a typical session requires.

### Response: TurnDepthRow

<ResponseField name="avg_turns_per_session" type="number">
  Mean number of turns per session across the window.
</ResponseField>

<ResponseField name="single_turn_rate_pct" type="number">
  Percentage of sessions resolved in a single turn.
</ResponseField>

<ResponseField name="p50_turns" type="number">
  Median turns per session.
</ResponseField>

<ResponseField name="p90_turns" type="number">
  90th percentile turns per session.
</ResponseField>

<ResponseField name="p95_turns" type="number">
  95th percentile turns per session.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/analytics/sessions/turn-depth?start_date=2024-06-01&end_date=2024-06-30" \
  -H "x-api-key: <key>"
```

```json Response
{
  "avg_turns_per_session": 4.2,
  "single_turn_rate_pct": 18.5,
  "p50_turns": 3,
  "p90_turns": 9,
  "p95_turns": 14
}
```
</CodeGroup>
