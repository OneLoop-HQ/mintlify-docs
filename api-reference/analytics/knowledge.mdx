## Overview

Knowledge analytics endpoints provide visibility into the health and utilization of your knowledge bases ŌĆö from how quickly documents are processed to whether your agents are successfully retrieving relevant content. All endpoints live under `/v1/analytics/knowledge`.

---

## Document Adoption

```
GET /v1/analytics/knowledge/documents/adoption
```

Returns KB adoption metrics ŌĆö what proportion of indexed documents are actively retrieved and contributing to agent responses.

### Response

<ResponseField name="kb_adoption_rate" type="number">
  Fraction of knowledge bases that have been retrieved from at least once in the window.
</ResponseField>

<ResponseField name="documents_active" type="integer">
  Count of documents retrieved at least once in the reporting window.
</ResponseField>

---

## Ingestion Timeseries

```
GET /v1/analytics/knowledge/ingestion-timeseries
```

Returns document ingestion (indexing) events over time.

### Query Parameters

<ParamField query="start_date" type="string" required>
  Start date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="end_date" type="string" required>
  End date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="granularity" type="string">
  `hour`, `day`, or `week`. Default: `day`.
</ParamField>

### Response

Each row: `time_bucket`, `documents_indexed`, `documents_failed`, `documents_quarantined`.

---

## Processing Latency

```
GET /v1/analytics/knowledge/processing-latency
```

Returns p50, p90, p95, and p99 latency for document processing (from upload to indexed state) in seconds.

---

## Cleanup Summary

```
GET /v1/analytics/knowledge/cleanup-summary
```

Returns counts of documents removed, expired, or archived in the reporting window.

---

## Retrieval Timeseries

```
GET /v1/analytics/knowledge/retrieval-timeseries
```

Returns KB retrieval metrics over time ŌĆö including hits, misses, and hit rate.

### Response fields per row

<ResponseField name="time_bucket" type="string (ISO 8601)">
  Start of the time bucket.
</ResponseField>

<ResponseField name="total_retrievals" type="integer">
  Total retrieval attempts.
</ResponseField>

<ResponseField name="hits" type="integer">
  Retrievals that returned at least one relevant document.
</ResponseField>

<ResponseField name="misses" type="integer">
  Retrievals that returned no relevant documents.
</ResponseField>

<ResponseField name="hit_rate_pct" type="number">
  Percentage of retrievals that were hits.
</ResponseField>

---

## Retrieval By Agent

```
GET /v1/analytics/knowledge/retrieval-by-agent
```

Returns retrieval hit rate and volume broken down by agent. Useful for identifying agents with low KB utilization.

---

## Top Retrieved Documents

```
GET /v1/analytics/knowledge/top-retrieved
```

Returns the most frequently retrieved documents in the window, ordered by retrieval count descending.

### Response fields per row

<ResponseField name="document_id" type="string">
  Unique document identifier.
</ResponseField>

<ResponseField name="document_title" type="string">
  Document title or filename.
</ResponseField>

<ResponseField name="retrieval_count" type="integer">
  Number of times retrieved.
</ResponseField>

<ResponseField name="citation_count" type="integer">
  Number of times cited in an agent response.
</ResponseField>

---

## Stale Documents

```
GET /v1/analytics/knowledge/stale
```

Returns documents that have not been retrieved in the reporting window. Use this to identify content that may be outdated, irrelevant, or poorly indexed.

---

## Knowledge Base Inventory

```
GET /v1/analytics/knowledge/inventory
```

Returns a comprehensive snapshot of what's in your knowledge bases right now.

### Response: InventoryEnvelope

<ResponseField name="summary" type="object">
  Headline counts:
  - `total_documents` ŌĆö Total indexed documents.
  - `total_chunks` ŌĆö Total text chunks across all documents.
  - `total_tokens` ŌĆö Estimated total token count.
  - `documents_added_last_7d` ŌĆö Documents indexed in the last 7 days.
  - `documents_removed_last_7d` ŌĆö Documents removed in the last 7 days.
</ResponseField>

<ResponseField name="by_source_type" type="array of objects">
  Document counts grouped by source type (e.g. `file_upload`, `web_connector`, `api_push`). Each entry: `source_type`, `count`.
</ResponseField>

<ResponseField name="by_hash_method" type="array of objects">
  Document counts grouped by deduplication hash method. Each entry: `hash_method`, `count`.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/analytics/knowledge/inventory \
  -H "x-api-key: <key>"
```

```json Response
{
  "summary": {
    "total_documents": 4820,
    "total_chunks": 38100,
    "total_tokens": 9400000,
    "documents_added_last_7d": 214,
    "documents_removed_last_7d": 12
  },
  "by_source_type": [
    { "source_type": "file_upload", "count": 3100 },
    { "source_type": "web_connector", "count": 1720 }
  ],
  "by_hash_method": [
    { "hash_method": "sha256", "count": 4820 }
  ]
}
```
</CodeGroup>

## Connector Freshness

```
GET /v1/analytics/knowledge/connectors/freshness
```

Returns per-connector sync status ŌĆö when each knowledge source was last synced and how stale it is.

### Response fields per connector

<ResponseField name="connector_id" type="string">
  Unique connector identifier.
</ResponseField>

<ResponseField name="connector_name" type="string">
  Human-readable connector name.
</ResponseField>

<ResponseField name="last_sync_status" type="string">
  `success`, `partial`, `failed`, or `never`.
</ResponseField>

<ResponseField name="last_synced_at" type="string (ISO 8601)">
  Timestamp of the last successful sync.
</ResponseField>

<ResponseField name="time_since_last_event_seconds" type="integer">
  Seconds since the last sync event of any kind.
</ResponseField>

## Dashboard (All-in-One)

```
GET /v1/analytics/knowledge/dashboard
```

Returns all key knowledge analytics in one request: adoption metrics, ingestion timeseries, processing latency, cleanup summary, retrieval timeseries, inventory snapshot, and connector freshness.

### Query Parameters

<ParamField query="start_date" type="string" required>
  Start date in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="end_date" type="string" required>
  End date in `YYYY-MM-DD` format.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/analytics/knowledge/dashboard?start_date=2024-06-01&end_date=2024-06-30" \
  -H "x-api-key: <key>"
```
</CodeGroup>
