## Overview

Feather's privacy layer automatically detects and scrubs personally identifiable information (PII) from conversation turns, memory syncs, and voice call transcripts. You control which entity types are scrubbed, define custom allow/deny patterns, and configure audio redaction behavior for voice channels.

---

## Get Privacy Config

```
GET /v1/privacy/config
```

Returns the current organization-wide privacy configuration.

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/privacy/config \
  -H "x-api-key: <key>"
```
</CodeGroup>

## Create Privacy Config

```
POST /v1/privacy/config
```

Creates the privacy configuration. If a config already exists, use `PATCH` to update it.

### Request Body

<ParamField body="pii_scrubbing_enabled" type="boolean" required>
  Master switch for PII scrubbing. When `false`, no scrubbing occurs regardless of other settings.
</ParamField>

<ParamField body="scrub_types" type="array of strings">
  Entity types to detect and scrub. Default built-in types include:

  - `ssn` — US Social Security Numbers
  - `credit_card` — Credit and debit card numbers
  - `iban` — International Bank Account Numbers
  - `us_bank_number` — US routing and account numbers
  - `email_address`, `phone_number`, `ip_address`, `date_of_birth`, `passport`, `driver_license`
</ParamField>

<ParamField body="custom_deny_patterns" type="array of objects">
  Custom regex patterns to detect and redact. Each entry:
  - `name` (string) — Friendly label for this pattern.
  - `regex` (string) — PCRE-compatible regular expression.
  - `replacement_label` (string) — Text to substitute in place of the match, e.g. `[EMPLOYEE_ID]`.
  - `score` (number, 0–1) — Confidence threshold required to trigger a scrub for this pattern.
</ParamField>

<ParamField body="custom_allow_patterns" type="array of objects">
  Patterns to explicitly exempt from scrubbing (allow-list). Each entry:
  - `name` (string) — Friendly label.
  - `regex` (string) — Matches to this pattern will not be scrubbed.
</ParamField>

<ParamField body="score_threshold" type="number">
  Global confidence threshold (0.0–1.0) for built-in detectors. Entities detected below this score are not scrubbed. Default: `0.5`.
</ParamField>

<ParamField body="audio_scrubbing_enabled" type="boolean">
  When `true`, voice call audio is processed for PII redaction in addition to transcripts.
</ParamField>

<ParamField body="audio_retention_policy" type="string">
  Controls what audio is stored after a call:
  - `transcript_only` — Store only the text transcript; no audio retained.
  - `redacted_audio_retained` — Store audio with PII segments replaced.
  - `raw` — Store unmodified audio (not recommended for compliance-sensitive deployments).
</ParamField>

<ParamField body="audio_redaction_fill" type="string">
  How PII segments are replaced in redacted audio:
  - `silence` — Replace with silence.
  - `tone_1khz` — Replace with a 1 kHz tone.
</ParamField>

<ParamField body="supported_languages" type="array of strings">
  ISO 639-1 language codes for which PII detection is enabled, e.g. `["en", "es", "fr"]`.
</ParamField>

<ParamField body="default_language" type="string">
  Fallback language code used when `language_detection_enabled` is `false`.
</ParamField>

<ParamField body="language_detection_enabled" type="boolean">
  When `true`, Feather auto-detects the language of each turn before applying PII detection.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/privacy/config \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "pii_scrubbing_enabled": true,
    "scrub_types": ["ssn", "credit_card", "iban", "us_bank_number", "email_address", "phone_number"],
    "score_threshold": 0.6,
    "custom_deny_patterns": [
      {
        "name": "Internal Employee ID",
        "regex": "EMP-\\d{6}",
        "replacement_label": "[EMPLOYEE_ID]",
        "score": 0.9
      }
    ],
    "custom_allow_patterns": [],
    "audio_scrubbing_enabled": true,
    "audio_retention_policy": "redacted_audio_retained",
    "audio_redaction_fill": "silence",
    "supported_languages": ["en", "es"],
    "default_language": "en",
    "language_detection_enabled": true
  }'
```
</CodeGroup>

## Update Privacy Config (Partial)

```
PATCH /v1/privacy/config
```

Partially updates the privacy configuration. Only fields included in the request body are changed; all others remain unchanged.

### Example

<CodeGroup>
```bash cURL
curl -X PATCH https://api-sandbox.featherhq.com/v1/privacy/config \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{"score_threshold": 0.75}'
```
</CodeGroup>

---

## Delete Privacy Config

```
DELETE /v1/privacy/config
```

Resets the privacy configuration to platform defaults. PII scrubbing remains enabled at the default threshold, with only the built-in `ssn`, `credit_card`, `iban`, and `us_bank_number` types active.

## PII Audit Logs

```
GET /v1/privacy/audit-logs
```

Returns a paginated log of every scrubbing event, showing where PII was detected and what action was taken.

### Query Parameters

<ParamField query="from" type="string (ISO 8601)">
  Start of the time range (inclusive).
</ParamField>

<ParamField query="to" type="string (ISO 8601)">
  End of the time range (inclusive).
</ParamField>

<ParamField query="source" type="string">
  Filter by event source:
  - `conversation_turn` — PII found in a chat or SMS message.
  - `memory_sync` — PII found during a memory write.
  - `voice_call` — PII found in a voice transcript or audio.
</ParamField>

<ParamField query="entity_type" type="string">
  Filter to events for a specific entity type, e.g. `credit_card`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/privacy/audit-logs?source=conversation_turn&from=2024-06-01T00:00:00Z&to=2024-06-30T23:59:59Z" \
  -H "x-api-key: <key>"
```
</CodeGroup>

## PII Audit Statistics

```
GET /v1/privacy/audit-stats
```

Returns aggregated scrubbing statistics over a time range, useful for compliance reporting and trend analysis.

### Query Parameters

<ParamField query="from" type="string (ISO 8601)" required>
  Start of the reporting window.
</ParamField>

<ParamField query="to" type="string (ISO 8601)" required>
  End of the reporting window.
</ParamField>

<ParamField query="bucket" type="string">
  Aggregation granularity: `day` or `hour`. Default: `day`.
</ParamField>

### Response

<ResponseField name="total_events" type="integer">
  Total number of scrubbing events in the window.
</ResponseField>

<ResponseField name="total_detections" type="integer">
  Total number of individual PII entities detected.
</ResponseField>

<ResponseField name="by_source" type="object">
  Detections broken down by source (`conversation_turn`, `memory_sync`, `voice_call`).
</ResponseField>

<ResponseField name="by_entity_type" type="array of objects">
  Each entry has `entity_type` and `count`, ordered by frequency descending.
</ResponseField>

<ResponseField name="trend" type="array of objects">
  Time-series breakdown bucketed by the `bucket` param. Each entry: `time_bucket`, `total_detections`.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/privacy/audit-stats?from=2024-06-01T00:00:00Z&to=2024-06-30T23:59:59Z&bucket=day" \
  -H "x-api-key: <key>"
```
