API keys are the primary authentication mechanism for the Feather platform. Each key belongs to your organization and carries a set of scopes that determine what it can do. Only `org:admin` members can create or revoke keys.

<Warning>
  The full plain-text API key is returned **only once** — in the response to the create request. Feather stores only a hashed version. If you lose the key, you must revoke it and create a new one. Store it immediately in a secret manager.
</Warning>

---

## GET /v1/identity/api-keys

List all active API keys in your organization. Key values are never returned in list or get responses — only metadata is shown.

```
GET https://api-sandbox.featherhq.com/v1/identity/api-keys
```

### Example Request

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://api-sandbox.featherhq.com/v1/identity/api-keys \
  --header 'x-api-key: YOUR_API_KEY'
```

```python Python
import httpx

res = httpx.get(
    "https://api-sandbox.featherhq.com/v1/identity/api-keys",
    headers={"x-api-key": "YOUR_API_KEY"},
)
print(res.json())
```

</CodeGroup>

### Response Fields (`ApiKeyResponse`)

<ResponseField name="id" type="string (uuid)" required>
  The unique identifier for this API key. Use this as `key_id` when revoking.
</ResponseField>

<ResponseField name="name" type="string" required>
  Human-readable label assigned to the key at creation.
</ResponseField>

<ResponseField name="scopes" type="array of strings" required>
  The permission scopes granted to this key. See [Scopes](#scopes) below.
</ResponseField>

<ResponseField name="key_prefix" type="string" required>
  The first few characters of the key (e.g., `fth_live_Ab1C`), useful for identifying a key without exposing its full value.
</ResponseField>

<ResponseField name="expires_at" type="string (ISO 8601) | null">
  Expiry timestamp, or `null` if the key does not expire.
</ResponseField>

<ResponseField name="created_at" type="string (ISO 8601)" required>
  When the key was created.
</ResponseField>

<ResponseField name="last_used_at" type="string (ISO 8601) | null">
  Timestamp of the most recent successful API call using this key.
</ResponseField>

### Example Response

```json
[
  {
    "id": "b3c4d5e6-1111-4abc-8888-ffeeddccbbaa",
    "name": "Production Integration",
    "scopes": ["conversations:read", "conversations:write", "agents:read"],
    "key_prefix": "fth_live_Ab1C",
    "expires_at": null,
    "created_at": "2024-01-15T09:00:00Z",
    "last_used_at": "2024-06-01T17:42:00Z"
  }
]
```

## POST /v1/identity/api-keys

Create a new API key. The `plain_text_key` in the response is the only time the full key value is available.

```
POST https://api-sandbox.featherhq.com/v1/identity/api-keys
```

### Request Body

<ParamField body="name" type="string" required>
  A descriptive label for the key. Choose something that identifies the system or integration it will be used by (e.g., `"CRM Webhook Integration"`).
</ParamField>

<ParamField body="scopes" type="array of strings" required>
  The permission scopes to grant this key. At least one scope is required. See [Scopes](#scopes) for valid values.
</ParamField>

<ParamField body="expires_at" type="string (ISO 8601)">
  Optional expiry datetime. If omitted, the key does not expire. Recommended for short-lived integrations.
</ParamField>

### Response Fields (`ApiKeyCreateResponse`)

<ResponseField name="id" type="string (uuid)" required>
  The key's unique identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  The name assigned to this key.
</ResponseField>

<ResponseField name="scopes" type="array of strings" required>
  The scopes granted to this key.
</ResponseField>

<ResponseField name="plain_text_key" type="string" required>
  **The full API key value.** This is returned exactly once. Copy it to a secure secret store immediately — it cannot be retrieved again.
</ResponseField>

<ResponseField name="key_prefix" type="string" required>
  A short prefix of the key for identification purposes.
</ResponseField>

<ResponseField name="expires_at" type="string (ISO 8601) | null">
  Expiry timestamp or `null`.
</ResponseField>

<ResponseField name="created_at" type="string (ISO 8601)" required>
  Creation timestamp.
</ResponseField>

### Example Request

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api-sandbox.featherhq.com/v1/identity/api-keys \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Production Integration",
    "scopes": ["conversations:read", "conversations:write", "agents:read"],
    "expires_at": "2025-12-31T23:59:59Z"
  }'
```

```python Python
import httpx

res = httpx.post(
    "https://api-sandbox.featherhq.com/v1/identity/api-keys",
    headers={"x-api-key": "YOUR_API_KEY"},
    json={
        "name": "Production Integration",
        "scopes": ["conversations:read", "conversations:write", "agents:read"],
        "expires_at": "2025-12-31T23:59:59Z",
    },
)
data = res.json()
# Store data["plain_text_key"] in your secret manager NOW
print(data["plain_text_key"])
```

</CodeGroup>

### Example Response

```json
{
  "id": "b3c4d5e6-1111-4abc-8888-ffeeddccbbaa",
  "name": "Production Integration",
  "scopes": ["conversations:read", "conversations:write", "agents:read"],
  "plain_text_key": "fth_live_Ab1Cxyz9876secretvaluehere",
  "key_prefix": "fth_live_Ab1C",
  "expires_at": "2025-12-31T23:59:59Z",
  "created_at": "2024-06-01T12:00:00Z"
}
```

## DELETE /v1/identity/api-keys/{key_id}

Revoke an API key immediately. Any in-flight requests using this key will fail after revocation.

```
DELETE https://api-sandbox.featherhq.com/v1/identity/api-keys/{key_id}
```

### Path Parameters

<ParamField path="key_id" type="string (uuid)" required>
  The `id` of the key to revoke. Retrieve this from the list endpoint.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url https://api-sandbox.featherhq.com/v1/identity/api-keys/b3c4d5e6-1111-4abc-8888-ffeeddccbbaa \
  --header 'x-api-key: YOUR_API_KEY'
```

</CodeGroup>

Returns `204 No Content` on success.

<Note>
  You cannot revoke the API key you are currently using to make the request. Use a different admin key to perform key rotation.
</Note>

## Scopes

Scopes control what a given API key is authorized to do. Assign the minimum required scopes for each integration.

| Scope | Description |
|---|---|
| `conversations:read` | Read conversation history and session data. |
| `conversations:write` | Create and update conversations, send messages. |
| `agents:read` | Read agent configurations and revisions. |
| `agents:write` | Create and update agents, revisions, and policies. |
| `identity:read` | Read end users and organization members. |
| `identity:write` | Create, update, and delete end users and contacts. |
| `knowledge:read` | Read knowledge base articles. |
| `knowledge:write` | Create and update knowledge base content. |
| `admin` | Full access to all endpoints including billing and key management. |

<Tip>
  For webhook-receiving integrations that only need to look up conversation context, assign `conversations:read` and `identity:read` only. Narrowly scoped keys limit blast radius if a key is compromised.
</Tip>
