Policies govern what an agent is allowed to say and do. They are enforced at specific **enforcement points** during a conversation — such as before a tool is called, after a response is generated, or when sensitive topics are detected. Attaching a policy to an agent revision binds its rules to that revision's runtime.

You can enable or disable all policy enforcement with a single `policies_enabled` flag, without removing individual policy attachments.

---

## GET /v1/agents/revisions/{id}/policies

Retrieve all policies currently attached to an agent revision, along with enforcement metadata.

```
GET https://api-sandbox.featherhq.com/v1/agents/revisions/{id}/policies
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent revision UUID.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/policies \
  --header 'x-api-key: YOUR_API_KEY'
```

```python Python
import httpx

res = httpx.get(
    "https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/policies",
    headers={"x-api-key": "YOUR_API_KEY"},
)
print(res.json())
```

</CodeGroup>

### Response Fields (`PolicyAttachmentResponse`)

<ResponseField name="revision_id" type="string (uuid)" required>
  The agent revision these policies are attached to.
</ResponseField>

<ResponseField name="policies_enabled" type="boolean" required>
  Master switch for policy enforcement. When `false`, policies are attached but not enforced at runtime.
</ResponseField>

<ResponseField name="policy_refs" type="array of objects" required>
  <Expandable title="PolicyRefSpec">
    <ResponseField name="policy_id" type="string (uuid)" required>
      The UUID of the attached policy.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="inert_tool_policies" type="array of objects">
  <Expandable title="InertToolPolicy">
    <ResponseField name="policy_id" type="string (uuid)">
      UUID of the policy that is inert on this revision.
    </ResponseField>
    <ResponseField name="policy_name" type="string">
      Human-readable name of the policy.
    </ResponseField>
    <ResponseField name="reason" type="string">
      Explanation of why the policy is inert (e.g., `"Policy targets tool 'billing_lookup' which is not present in this revision"`).
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Response

```json
{
  "revision_id": "r1a2b3c4-0000-4def-9000-112233445566",
  "policies_enabled": true,
  "policy_refs": [
    { "policy_id": "p1a2b3c4-0000-4abc-1111-aaaabbbbcccc" },
    { "policy_id": "p2b3c4d5-0000-4bcd-2222-bbbbccccdddd" }
  ],
  "inert_tool_policies": [
    {
      "policy_id": "p3c4d5e6-0000-4cde-3333-ccccddddeee",
      "policy_name": "Billing Refund Guard",
      "reason": "Policy targets tool 'billing_refund_tool' which is not present in this revision."
    }
  ]
}
```

<Warning>
  **Review `inert_tool_policies` carefully.** A policy listed here has no effect at runtime because the tool it targets is not attached to this revision. This may indicate a misconfiguration — either the intended tool is missing from the revision, or the policy was copied from a different agent and needs to be updated.
</Warning>

## PUT /v1/agents/revisions/{id}/policies

Fully replace the set of policies attached to an agent revision. All existing policy attachments are removed and replaced with the list you provide. To toggle enforcement without changing attachments, omit `policy_refs` and send only `policies_enabled`.

```
PUT https://api-sandbox.featherhq.com/v1/agents/revisions/{id}/policies
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent revision UUID.
</ParamField>

### Request Body

<ParamField body="policy_refs" type="array of objects" required>
  The complete new set of policy attachments. This is a **full replace** — any policies not listed here will be detached.

  <Expandable title="PolicyRefSpec">
    <ParamField body="policy_id" type="string (uuid)" required>
      UUID of the policy to attach.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="policies_enabled" type="boolean">
  Set to `true` to enforce all attached policies, or `false` to suspend enforcement without removing attachments. Defaults to `true` if omitted.
</ParamField>

### Example Request — Replace All Policies

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/policies \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "policy_refs": [
      { "policy_id": "p1a2b3c4-0000-4abc-1111-aaaabbbbcccc" },
      { "policy_id": "p4d5e6f7-0000-4def-4444-ddddeeeeffff" }
    ],
    "policies_enabled": true
  }'
```

</CodeGroup>

### Example Request — Disable All Policies (Without Detaching)

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/policies \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "policy_refs": [
      { "policy_id": "p1a2b3c4-0000-4abc-1111-aaaabbbbcccc" },
      { "policy_id": "p2b3c4d5-0000-4bcd-2222-bbbbccccdddd" }
    ],
    "policies_enabled": false
  }'
```

</CodeGroup>

Returns the updated `PolicyAttachmentResponse`.

## DELETE /v1/agents/revisions/{id}/policies

Detach **all** policies from an agent revision. This is equivalent to calling PUT with an empty `policy_refs` array.

```
DELETE https://api-sandbox.featherhq.com/v1/agents/revisions/{id}/policies
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent revision UUID.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/policies \
  --header 'x-api-key: YOUR_API_KEY'
```

</CodeGroup>

Returns `204 No Content` on success.

<Note>
  Detaching all policies removes the attachments from this revision only. The policy definitions themselves are not deleted and can be re-attached or used by other revisions.
</Note>

## GET /v1/agents/{id}/enforcement-plan

Returns a detailed description of how each attached policy will be enforced at runtime for the agent's **currently active revision**. Use this to audit policy behavior before promoting a revision or to debug unexpected enforcement outcomes.

```
GET https://api-sandbox.featherhq.com/v1/agents/{id}/enforcement-plan
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent UUID. The enforcement plan is computed for the active revision.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://api-sandbox.featherhq.com/v1/agents/9a1b2c3d-0000-4abc-8000-aabbccddeeff/enforcement-plan \
  --header 'x-api-key: YOUR_API_KEY'
```

</CodeGroup>

### Response Fields

<ResponseField name="agent_id" type="string (uuid)" required>
  The agent UUID.
</ResponseField>

<ResponseField name="revision_id" type="string (uuid)" required>
  The active revision UUID the plan was computed for.
</ResponseField>

<ResponseField name="policies_enabled" type="boolean" required>
  Whether policy enforcement is active on this revision.
</ResponseField>

<ResponseField name="enforcement_points" type="array of objects">
  <Expandable title="EnforcementPoint">
    <ResponseField name="point" type="string" required>
      The enforcement point in the conversation lifecycle. Examples: `pre_tool_call`, `post_response`, `topic_detection`, `escalation`.
    </ResponseField>
    <ResponseField name="policies" type="array of objects">
      <Expandable title="PolicyEnforcementEntry">
        <ResponseField name="policy_id" type="string (uuid)">
          UUID of the policy enforced at this point.
        </ResponseField>
        <ResponseField name="policy_name" type="string">
          Human-readable policy name.
        </ResponseField>
        <ResponseField name="action" type="string">
          The enforcement action taken when the policy is triggered. Examples: `block`, `warn`, `redact`, `escalate`.
        </ResponseField>
        <ResponseField name="is_active" type="boolean">
          Whether this policy is currently active (i.e., not inert and `policies_enabled` is `true`).
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Response

```json
{
```

<Tip>
  Pull the enforcement plan after every revision activation to confirm that all expected policies are active and no policies are listed as inert. This is especially important when copying a revision from one agent to another, as tool assignments may differ.
</Tip>
