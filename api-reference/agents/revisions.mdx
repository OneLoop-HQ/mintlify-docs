---
title: "Agent Revisions: Version Control for Agent Behavior"
sidebarTitle: "Revisions"
description: "Create, update, and set active agent revisions. Revisions control the system prompt, tools, knowledge bases, policies, and model settings."
---

Revisions are versioned snapshots of an agent's behavioral configuration. Every change to a system prompt, tool assignment, knowledge base binding, or model setting is captured in a revision. Only one revision is "active" at a time for live traffic — all others remain as history you can audit or roll back to.

This separation of **agent metadata** (name, description) from **agent behavior** (revision) allows safe iteration without interrupting live conversations.

---

## POST /v1/agents/{id}/revisions

Create a new revision for an agent. The revision starts in `draft` status and does not affect live traffic until you call [set-active-revision](#post-v1agentsidset-active-revision).

```
POST https://api-sandbox.featherhq.com/v1/agents/{id}/revisions
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent UUID to create a revision for.
</ParamField>

### Request Body

<ParamField body="name" type="string" required>
  A label for this revision (e.g., `"v3 — Add billing tool"`).
</ParamField>

<ParamField body="description" type="string">
  Notes on what changed in this revision. Useful for changelog tracking.
</ParamField>

<ParamField body="system_prompt" type="string">
  The full system prompt for this revision. Supports Feather template variables.
</ParamField>

<ParamField body="tool_refs" type="array of objects">
  <Expandable title="ToolRef">
    <ParamField body="tool_id" type="string (uuid)" required>
      UUID of the tool to attach.
    </ParamField>
    <ParamField body="kind" type="string" required>
      Tool category. One of:
      - `api` — External HTTP API call tool.
      - `integration` — Managed integration (e.g., Salesforce, Zendesk).
      - `util` — Built-in utility (e.g., datetime, math, web search).
      - `kb` — Knowledge base search tool.
    </ParamField>
    <ParamField body="enabled" type="boolean">
      Whether this tool is active in this revision. Default: `true`.
    </ParamField>
    <ParamField body="priority" type="integer">
      Ordering hint when multiple tools could satisfy a request.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="knowledge_base_refs" type="array of objects">
  <Expandable title="KnowledgeBaseRef">
    <ParamField body="knowledge_base_id" type="string (uuid)" required>
      UUID of the knowledge base to bind.
    </ParamField>
    <ParamField body="enabled" type="boolean">
      Whether this KB is searchable in this revision. Default: `true`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="policy_refs" type="array of objects">
  <Expandable title="PolicyRefSpec">
    <ParamField body="policy_id" type="string (uuid)" required>
      UUID of the policy to attach.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="policies_enabled" type="boolean">
  Master switch for policy enforcement on this revision. When `false`, all attached policies are suspended. Default: `true`.
</ParamField>

<ParamField body="channel_specific_config" type="object">
  Per-channel configuration overrides. Keys are channel names (`voice`, `chat`, `sms`, `email`). Values are channel config objects with fields like `greeting_message` or `max_response_tokens`.
</ParamField>

<ParamField body="router_config_id" type="string (uuid)">
  Override the agent-level router config for this revision.
</ParamField>

<ParamField body="model_settings" type="object">
  <Expandable title="ModelSettings">
    <ParamField body="model_id" type="string">
      LLM model identifier (e.g., `"gpt-4o"`, `"claude-3-5-sonnet"`).
    </ParamField>
    <ParamField body="temperature" type="number">
      Sampling temperature `0.0`–`2.0`. Default: `0.7`.
    </ParamField>
    <ParamField body="max_tokens" type="integer">
      Maximum response tokens. Default: `1024`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="kb_clearance_level" type="integer">
  KB clearance level override for this revision (0–100).
</ParamField>

<ParamField body="kb_audience_tags" type="array of strings">
  Audience tags used to filter KB articles for this revision.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api-sandbox.featherhq.com/v1/agents/9a1b2c3d-0000-4abc-8000-aabbccddeeff/revisions \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "v3 — Add billing tool",
    "description": "Adds the billing lookup tool and upgrades to gpt-4o.",
    "system_prompt": "You are a support agent for {{brand_name}}. You can look up billing records.",
    "tool_refs": [
      { "tool_id": "t1a2b3c4-0000-4abc-aaaa-111111111111", "kind": "api", "enabled": true },
      { "tool_id": "t2b3c4d5-0000-4def-bbbb-222222222222", "kind": "kb", "enabled": true }
    ],
    "knowledge_base_refs": [
      { "knowledge_base_id": "kb1a2b3c-0000-4abc-cccc-333333333333", "enabled": true }
    ],
    "policies_enabled": true,
    "model_settings": { "model_id": "gpt-4o", "temperature": 0.5, "max_tokens": 1500 },
    "kb_clearance_level": 50
  }'
```

```python Python

res = httpx.post(
    "https://api-sandbox.featherhq.com/v1/agents/9a1b2c3d-0000-4abc-8000-aabbccddeeff/revisions",
```

## PUT /v1/agents/revisions/{id}

Update a draft revision. Active revisions cannot be modified — create a new revision and promote it instead.

```
PUT https://api-sandbox.featherhq.com/v1/agents/revisions/{id}
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The revision UUID. Must be in `draft` status.
</ParamField>

Accepts the same fields as the create endpoint. Only included fields are updated.

### Example Request

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566 \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "v3 — Add billing tool (reviewed)",
    "model_settings": { "temperature": 0.3 }
  }'
```

</CodeGroup>


### Path Parameters

  The revision UUID. Must be in `draft` status.

## POST /v1/agents/{id}/set-active-revision

Promote a draft revision to `active`, making it the revision used for all live conversations on this agent.

```
POST https://api-sandbox.featherhq.com/v1/agents/{id}/set-active-revision
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent UUID.
</ParamField>

### Request Body

<ParamField body="revision_id" type="string (uuid)" required>
  The UUID of the draft revision to activate.
</ParamField>

<ParamField body="bypass_release_gate" type="boolean">
  When `true`, skips any configured release gate checks. Requires `org:admin` role. Default: `false`.
</ParamField>

<ParamField body="override_reason" type="string">
  Required when `bypass_release_gate` is `true`. A human-readable explanation recorded in the audit log.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api-sandbox.featherhq.com/v1/agents/9a1b2c3d-0000-4abc-8000-aabbccddeeff/set-active-revision \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "revision_id": "r1a2b3c4-0000-4def-9000-112233445566",
    "bypass_release_gate": false
  }'
```

</CodeGroup>

Returns the updated `AgentResponse` with `active_revision_id` set to the promoted revision.

<Warning>
  Promoting a revision immediately affects all new conversations. In-flight conversations may continue using the previously active revision until they end, depending on your session continuity configuration.
</Warning>

<Tip>
  Use the `override_reason` field when bypassing release gates. This creates a permanent audit trail and is required by some compliance frameworks. Examples: `"Hotfix for incorrect phone number format"`.
</Tip>

## Tool Ref Kinds

| Kind | Description |
|---|---|
| `api` | Calls an external HTTP endpoint you have defined in the Tools section. |
| `integration` | Uses a Feather-managed integration connector (e.g., Zendesk, Salesforce, Stripe). |
| `util` | Built-in utility tools provided by Feather (e.g., `datetime`, `web_search`, `calculator`). |
| `kb` | Enables the agent to perform semantic search over an attached knowledge base. |
