---
title: "Attach Workflows to Agent Revisions"
sidebarTitle: "Agent Workflows"
description: "Attach and detach workflow revisions from agent revisions, control priority and enabled state."
---

Workflows are structured sequences of steps — branching logic, API calls, data lookups — that can be triggered within a conversation. Attaching a workflow to an agent revision makes it available for the agent to invoke. You can attach multiple workflows per revision and control the order in which they are considered using the `priority` field.

All endpoints operate on agent revisions, not agents directly. Make sure you are targeting the correct `revision_id`.

---

## POST /v1/agents/revisions/{id}/workflows

Attach a workflow to an agent revision. Optionally pin it to a specific workflow revision; if no `workflow_revision_id` is supplied, the workflow's current active revision is used at runtime.

```
POST https://api-sandbox.featherhq.com/v1/agents/revisions/{id}/workflows
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent revision UUID to attach the workflow to.
</ParamField>

### Request Body

<ParamField body="workflow_id" type="string (uuid)" required>
  The UUID of the workflow to attach.
</ParamField>

<ParamField body="workflow_revision_id" type="string (uuid)">
  Optional. Pin this attachment to a specific workflow revision. If omitted, the workflow's active revision is resolved at runtime.
</ParamField>

<ParamField body="priority" type="integer" required>
  Determines the order in which workflows are considered when multiple could be triggered. Lower numbers are evaluated first. Use `0` for the highest priority. Must be a non-negative integer.
</ParamField>

<ParamField body="enabled" type="boolean" required>
  Whether this workflow is active on this revision. Set to `false` to temporarily disable without detaching.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/workflows \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "workflow_id": "wf1a2b3c-0000-4abc-bbbb-aaaaaaaaaaaa",
    "priority": 0,
    "enabled": true
  }'
```

```python Python
import httpx

res = httpx.post(
    "https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/workflows",
    headers={"x-api-key": "YOUR_API_KEY"},
    json={
        "workflow_id": "wf1a2b3c-0000-4abc-bbbb-aaaaaaaaaaaa",
        "priority": 0,
        "enabled": True,
    },
)
print(res.json())
```

</CodeGroup>

### Response Fields (`AgentRevisionWorkflowResponse`)

<ResponseField name="workflow_id" type="string (uuid)" required>
  The UUID of the attached workflow.
</ResponseField>

<ResponseField name="workflow_name" type="string" required>
  Human-readable name of the workflow.
</ResponseField>

<ResponseField name="workflow_revision_id" type="string (uuid) | null">
  The pinned workflow revision UUID, or `null` if the active revision is resolved at runtime.
</ResponseField>

<ResponseField name="priority" type="integer" required>
  The evaluation priority for this workflow on this agent revision.
</ResponseField>

<ResponseField name="enabled" type="boolean" required>
  Whether this workflow is currently enabled.
</ResponseField>

<ResponseField name="created_at" type="string (ISO 8601)" required>
  When this workflow was attached to the revision.
</ResponseField>

## PUT /v1/agents/revisions/{id}/workflows/{workflow_id}

Update the priority, enabled state, or pinned revision for an attached workflow.

```
PUT https://api-sandbox.featherhq.com/v1/agents/revisions/{id}/workflows/{workflow_id}
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent revision UUID.
</ParamField>

<ParamField path="workflow_id" type="string (uuid)" required>
  The workflow UUID to update.
</ParamField>

### Request Body

<ParamField body="priority" type="integer">
  Updated evaluation priority. Lower values are evaluated first.
</ParamField>

<ParamField body="enabled" type="boolean">
  Toggle the workflow on or off without detaching it.
</ParamField>

<ParamField body="workflow_revision_id" type="string (uuid)">
  Pin or change the specific workflow revision. Set to `null` to revert to dynamic resolution of the active revision.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/workflows/wf2b3c4d-1111-4bcd-cccc-bbbbbbbbbbbb \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "enabled": true,
    "priority": 2
  }'
```

</CodeGroup>

Returns the updated `AgentRevisionWorkflowResponse`.

## DELETE /v1/agents/revisions/{id}/workflows/{workflow_id}

Detach a workflow from an agent revision.

```
DELETE https://api-sandbox.featherhq.com/v1/agents/revisions/{id}/workflows/{workflow_id}
```

### Path Parameters

<ParamField path="id" type="string (uuid)" required>
  The agent revision UUID.
</ParamField>

<ParamField path="workflow_id" type="string (uuid)" required>
  The workflow UUID to detach.
</ParamField>

### Example Request

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url https://api-sandbox.featherhq.com/v1/agents/revisions/r1a2b3c4-0000-4def-9000-112233445566/workflows/wf2b3c4d-1111-4bcd-cccc-bbbbbbbbbbbb \
  --header 'x-api-key: YOUR_API_KEY'
```

</CodeGroup>

Returns `204 No Content` on success.

<Note>
  Detaching a workflow from an agent revision only removes the association — the workflow itself is not deleted and remains available to attach to other revisions.
</Note>

<Tip>
  To temporarily disable a workflow without losing its configuration (priority, pinned revision), use the `PUT` endpoint with `"enabled": false` rather than deleting and re-attaching.
</Tip>
