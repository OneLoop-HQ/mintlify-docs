Ingestion converts your source documents into searchable vector embeddings stored in the knowledge base. You can ingest **files** (PDF, DOCX, TXT, Markdown, HTML) or **text snippets** directly — both in the same request.

Ingestion is **asynchronous**: the API immediately returns an `ingestion_job_id` and you poll the job endpoint for progress.

---

## Ingest Documents

```
POST https://api-sandbox.featherhq.com/v1/knowledge-base/knowledge-bases/{kb_id}/ingest
```

Accepts a `multipart/form-data` body with two parts:
1. **`manifest`** — A JSON array describing each document to ingest.
2. **`files`** — One or more binary file parts (required only for `type: file` manifest items).

<ParamField path="kb_id" type="string" required>
  Knowledge base ID to ingest into.
</ParamField>

### Manifest Item Fields

Each element in the `manifest` array describes one document.

<ParamField body="type" type="string" required>
  Document source type:
  - `file` — Binary file uploaded in the multipart body.
  - `text` — Plain text or Markdown content passed inline in the manifest.
</ParamField>

<ParamField body="file_index" type="integer">
  Zero-based index into the multipart `files` array. **Required** when `type` is `file`.
</ParamField>

<ParamField body="title" type="string" required>
  Document title used in citations and the document list.
</ParamField>

<ParamField body="content" type="string">
  Inline text content. **Required** when `type` is `text`.
</ParamField>

<ParamField body="document_id" type="string">
  If provided and matches an existing document ID in this knowledge base, the existing document will be **replaced** (re-ingested). Leave blank to create a new document.
</ParamField>

<ParamField body="sensitivity_level" type="string">
  Override the knowledge base's `default_sensitivity_level` for this document. One of `public`, `internal`, `confidential`, `restricted`.
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary key-value pairs stored on the document. Not used in retrieval ranking.
</ParamField>

### Response: IngestResponse

<ResponseField name="ingestion_job_id" type="string">
  ID of the background ingestion job. Use this to poll progress via [GET /ingestion-jobs/{id}](#get-an-ingestion-job).
</ResponseField>

<ResponseField name="items" type="array">
  Per-manifest-item results, in the same order as the input manifest.

  <Expandable title="item fields">
    <ResponseField name="manifest_index" type="integer">
      Zero-based index corresponding to the manifest array position.
    </ResponseField>
    <ResponseField name="result" type="string">
      - `queued` — Document has been accepted and queued for processing.
      - `duplicate` — An identical document already exists; ingestion was skipped.
    </ResponseField>
    <ResponseField name="document_id" type="string">
      The document ID assigned (new documents) or the existing document ID (duplicates or updates).
    </ResponseField>
  </Expandable>
</ResponseField>

## Complete curl Example

The following ingests two files and one inline text snippet in a single request:

```bash
curl -X POST \
  "https://api-sandbox.featherhq.com/v1/knowledge-base/knowledge-bases/kb_abc123/ingest" \
  -H "x-api-key: <key>" \
  -F 'manifest=[
    {
      "type": "file",
      "file_index": 0,
      "title": "Product Manual v3",
      "sensitivity_level": "internal"
    },
    {
      "type": "file",
      "file_index": 1,
      "title": "Refund Policy 2024",
      "metadata": {"department": "billing"}
    },
    {
      "type": "text",
      "title": "FAQ: How do I cancel?",
      "content": "To cancel your subscription, visit Account > Settings > Subscription and click Cancel. Cancellations take effect at the end of the billing period.",
      "sensitivity_level": "public"
    }
  ]' \
  -F "files=@/path/to/product_manual_v3.pdf" \
  -F "files=@/path/to/refund_policy_2024.docx"
```

<CodeGroup>
```python Python
import httpx, json

kb_id = "kb_abc123"
headers = {"x-api-key": "<key>"}

manifest = [
    {"type": "file", "file_index": 0, "title": "Product Manual v3"},
    {"type": "text", "title": "FAQ: Cancellation", "content": "To cancel, go to Account > Settings..."},
]

with open("product_manual_v3.pdf", "rb") as pdf:
    resp = httpx.post(
        f"https://api-sandbox.featherhq.com/v1/knowledge-base/knowledge-bases/{kb_id}/ingest",
        headers=headers,
        data={"manifest": json.dumps(manifest)},
        files=[("files", ("product_manual_v3.pdf", pdf, "application/pdf"))],
        timeout=60,
    )

data = resp.json()
job_id = data["ingestion_job_id"]
print(f"Job ID: {job_id}")
for item in data["items"]:
    print(f"  [{item['manifest_index']}] {item['result']} → {item['document_id']}")
```
</CodeGroup>

<Note>
  File size limits: individual files up to **50 MB**; total request body up to **200 MB**. For larger documents, split them into multiple ingest requests.
</Note>

## List Ingestion Jobs

```
GET https://api-sandbox.featherhq.com/v1/knowledge-base/knowledge-bases/{kb_id}/ingestion-jobs
```

Returns a cursor-paginated list of ingestion jobs for a knowledge base, ordered by `created_at` descending.

<ParamField path="kb_id" type="string" required>
  Knowledge base ID.
</ParamField>

<ParamField query="status" type="string">
  Filter by job status. See [Status Values](#ingestion-status-values).
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor.
</ParamField>

<ParamField query="limit" type="integer">
  Results per page. Default `20`, max `100`.
</ParamField>

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/knowledge-base/knowledge-bases/kb_abc123/ingestion-jobs?status=active" \
  -H "x-api-key: <key>"
```
</CodeGroup>

## Get an Ingestion Job

```
GET https://api-sandbox.featherhq.com/v1/knowledge-base/ingestion-jobs/{id}
```

Returns the current state of an ingestion job.

<ParamField path="id" type="string" required>
  Ingestion job ID.
</ParamField>

### IngestionJobResponse Fields

<ResponseField name="id" type="string">
  Ingestion job ID.
</ResponseField>

<ResponseField name="knowledge_base_id" type="string">
  Parent knowledge base ID.
</ResponseField>

<ResponseField name="status" type="string">
  Current job status. See [Status Values](#ingestion-status-values).
</ResponseField>

<ResponseField name="total_documents" type="integer">
  Total number of documents submitted in this job.
</ResponseField>

<ResponseField name="completed" type="integer">
  Number of documents successfully ingested.
</ResponseField>

<ResponseField name="failed" type="integer">
  Number of documents that failed ingestion.
</ResponseField>

<ResponseField name="progress_pct" type="number">
  Percentage complete: `(completed + failed) / total_documents * 100`. Range `0`–`100`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 job creation timestamp.
</ResponseField>

<ResponseField name="completed_at" type="string">
  ISO 8601 timestamp when the job reached a terminal state. `null` if still in progress.
</ResponseField>

### Ingestion Status Values

| Status | Description |
|---|---|
| `queued` | Job is waiting for a worker. No documents have started processing. |
| `processing` | At least one document is actively being chunked and embedded. |
| `active` | Shorthand for `queued` + `processing` — any non-terminal state. |
| `completed` | All documents processed successfully. |
| `partial_failure` | Job finished but some documents failed. `completed` + `failed` < `total_documents`. |
| `failed` | All documents failed or the job itself errored. |

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/knowledge-base/ingestion-jobs/ingjob_abc123 \
  -H "x-api-key: <key>"
```

```json Example response
{
  "id": "ingjob_abc123",
  "knowledge_base_id": "kb_abc123",
  "status": "partial_failure",
  "total_documents": 3,
  "completed": 2,
  "failed": 1,
  "progress_pct": 100.0,
  "created_at": "2024-06-01T10:00:00Z",
  "completed_at": "2024-06-01T10:02:34Z"
}
```

```python Python – poll until complete
import httpx, time

job_id = "ingjob_abc123"
headers = {"x-api-key": "<key>"}
base = "https://api-sandbox.featherhq.com"

while True:
    job = httpx.get(
        f"{base}/v1/knowledge-base/ingestion-jobs/{job_id}",
        headers=headers,
    ).json()

    print(f"Status: {job['status']}  Progress: {job['progress_pct']:.0f}%")

    if job["status"] not in ("queued", "processing", "active"):
        print(f"Done. Completed: {job['completed']}, Failed: {job['failed']}")
        break

    time.sleep(5)
```
</CodeGroup>

## Get Per-Document Status

```
GET https://api-sandbox.featherhq.com/v1/knowledge-base/ingestion-jobs/{id}/documents
```

Returns a paginated list of each document's ingestion result within a job.

<ParamField path="id" type="string" required>
  Ingestion job ID.
</ParamField>

<ParamField query="status" type="string">
  Filter by document status: `queued`, `processing`, `completed`, `failed`.
</ParamField>

<ResponseField name="items" type="array">
  Per-document status records.

  <Expandable title="item fields">
    <ResponseField name="document_id" type="string">Document ID.</ResponseField>
    <ResponseField name="title" type="string">Document title.</ResponseField>
    <ResponseField name="status" type="string">Individual document status.</ResponseField>
    <ResponseField name="chunk_count" type="integer">Number of chunks created. `0` until `completed`.</ResponseField>
    <ResponseField name="error_message" type="string">Error detail if `status` is `failed`.</ResponseField>
  </Expandable>
</ResponseField>

## Retry a Failed Job

```
POST https://api-sandbox.featherhq.com/v1/knowledge-base/ingestion-jobs/{id}/retry
```

Re-queues failed documents from the job for another ingestion attempt. Successfully completed documents are not re-processed.

<ParamField path="id" type="string" required>
  Ingestion job ID. Must have `status` of `failed` or `partial_failure`.
</ParamField>

### Response

Returns the updated **IngestionJobResponse** with `status` reset to `queued` or `processing` for the retried documents.

<CodeGroup>
```bash cURL
curl -X POST \
  https://api-sandbox.featherhq.com/v1/knowledge-base/ingestion-jobs/ingjob_abc123/retry \
  -H "x-api-key: <key>"
```
</CodeGroup>

<Tip>
  Before retrying, check the job logs (`GET /logs`) to understand why documents failed. Retrying without fixing the root cause (e.g. a corrupted PDF, an unsupported file format) will produce the same failure.
</Tip>
