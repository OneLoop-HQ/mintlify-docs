## Overview

`POST /v1/knowledge-base/search` performs a semantic search across one or more knowledge bases. Submit a natural-language query and receive ranked document chunks, an answerability assessment, and sensitivity metadata for each result.

**Base URL:** `https://api-sandbox.featherhq.com`

---

## Request

### Endpoint

```
POST /v1/knowledge-base/search
```

### Headers

| Header | Value |
|---|---|
| `x-api-key` | Your API key |
| `Content-Type` | `application/json` |

### Body Parameters

<ParamField body="query" type="string" required>
  The natural-language question or search phrase. Feather embeds this string and searches the vector index.
</ParamField>

<ParamField body="kb_ids" type="string[]" required>
  One or more knowledge base IDs to search. Passing multiple IDs performs a cross-KB search and returns a merged, ranked result set.
</ParamField>

<ParamField body="rerank" type="boolean" default="false">
  When `true`, a cross-encoder reranker re-scores each candidate chunk after initial retrieval. Improves precision at the cost of slightly higher latency.
</ParamField>

<ParamField body="limit" type="integer" default="10">
  Maximum number of chunks to return. Accepts values between `1` and `100` (inclusive).
</ParamField>

<ParamField body="prefer_draft" type="boolean" default="false">
  When `true`, draft (unpublished) document versions are preferred over published versions where both exist.
</ParamField>

<ParamField body="version_overrides" type="object">
  A map of `document_id → version_id`. Forces specific document versions into the search index for this request, regardless of the published state.

  ```json
  {
    "doc_abc123": "ver_xyz789"
  }
  ```
</ParamField>

<ParamField body="alpha" type="number" default="1.0">
  Controls the blend between dense (vector) and sparse (keyword) retrieval.

  | Value | Behaviour |
  |---|---|
  | `1.0` | Pure dense vector search |
  | `0.5` | Equal weight — hybrid search |
  | `0.0` | Pure sparse / keyword search |

  Values must be between `0.1` and `1.0`. Use `0.5`–`0.8` for most production workloads where recall matters.
</ParamField>

## Response

The response is a `SearchResponse` object.

### Top-Level Fields

<ResponseField name="results" type="SearchChunk[]">
  Flat, ranked list of matching chunks. Ordered by descending `score`.

  <Expandable title="SearchChunk fields">
    <ResponseField name="chunk_id" type="string">
      Unique identifier for this chunk within the index.
    </ResponseField>
    <ResponseField name="kb_id" type="string">
      The knowledge base this chunk belongs to.
    </ResponseField>
    <ResponseField name="document_id" type="string">
      Parent document identifier.
    </ResponseField>
    <ResponseField name="document_title" type="string">
      Human-readable title of the source document.
    </ResponseField>
    <ResponseField name="source_type" type="string">
      Origin of the document — e.g. `notion`, `google_drive`, `s3`, `url`, `upload`.
    </ResponseField>
    <ResponseField name="content" type="string">
      The raw text content of the chunk.
    </ResponseField>
    <ResponseField name="score" type="number">
      Relevance score between `0.0` and `1.0`. Higher is more relevant.
    </ResponseField>
    <ResponseField name="chunk_index" type="integer">
      Zero-based index of this chunk within the document.
    </ResponseField>
    <ResponseField name="position" type="object">
      Approximate character offsets within the source document: `{ start: number, end: number }`.
    </ResponseField>
    <ResponseField name="sensitivity" type="string">
      Human-readable sensitivity label: `public`, `internal`, `confidential`, or `restricted`.
    </ResponseField>
    <ResponseField name="sensitivity_level" type="integer">
      Numeric sensitivity tier. `0` = public, `1` = internal, `2` = confidential, `3` = restricted.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total_results" type="integer">
  Total number of chunks matched before the `limit` was applied.
</ResponseField>

<ResponseField name="grouped_results" type="object">
  Present only when the searched knowledge bases use different embedding spaces (incompatible vector models). Keys are `kb_id`; values are arrays of `SearchChunk`. Use this field instead of `results` when `ranking_mode` is `grouped_unranked`.
</ResponseField>

<ResponseField name="ranking_mode" type="string">
  Describes how results were ranked for this response.

  | Value | Meaning |
  |---|---|
  | `global_score` | Single unified score across all KBs |
  | `reranked` | Cross-encoder reranker was applied |
  | `grouped_unranked` | KBs have incompatible embedding spaces; results are grouped, not globally ranked |
</ResponseField>

<ResponseField name="answerability" type="object">
  High-level assessment of whether the query can be answered from the retrieved content.

  <Expandable title="answerability fields">
    <ResponseField name="label" type="string">
      One of `answerable`, `low_confidence`, `not_answerable`, or `unknown`.
    </ResponseField>
    <ResponseField name="reason_code" type="string">
      Machine-readable reason, e.g. `no_relevant_chunks`, `top_score_below_threshold`, `sufficient_coverage`.
    </ResponseField>
    <ResponseField name="raw_top_score" type="number">
      The highest relevance score among all retrieved chunks before any reranking.
    </ResponseField>
    <ResponseField name="score_basis" type="string">
      Which scoring step was used to compute answerability: `initial_retrieval` or `reranker`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/knowledge-base/search \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What is the refund policy for digital purchases?",
    "kb_ids": ["kb_01hx3rp2v9abcdef"],
    "rerank": true,
    "limit": 5,
    "alpha": 0.7
  }'
```

```python Python
import requests

response = requests.post(
    "https://api-sandbox.featherhq.com/v1/knowledge-base/search",
    headers={"x-api-key": "YOUR_API_KEY"},
    json={
        "query": "What is the refund policy for digital purchases?",
        "kb_ids": ["kb_01hx3rp2v9abcdef"],
        "rerank": True,
        "limit": 5,
        "alpha": 0.7,
    },
)
data = response.json()
```

## Understanding `alpha`

<Tip>
  Start with `alpha: 0.7` for most customer-support and FAQ workloads. Drop toward `0.5` when users ask precise keyword queries (e.g. product SKUs or order numbers).
</Tip>

The `alpha` parameter controls the balance between two retrieval strategies:

- **Dense (vector) retrieval** — embeds the query and finds semantically similar chunks. Best for paraphrased questions.
- **Sparse (keyword) retrieval** — uses BM25-style term matching. Best for exact terms and short queries.

```
alpha = 1.0  →  100 % dense
alpha = 0.5  →  50 % dense + 50 % sparse (hybrid)
alpha = 0.1  →  10 % dense + 90 % sparse
```

<Note>
  When searching across multiple knowledge bases with different embedding models, Feather automatically falls back to `grouped_unranked` mode regardless of the `alpha` value. Check the `ranking_mode` field in the response and switch to `grouped_results` when this occurs.
</Note>

## Answerability Labels

| Label | Meaning |
|---|---|
| `answerable` | Top-scoring chunk(s) strongly match the query |
| `low_confidence` | Some relevant content was found but coverage is incomplete |
| `not_answerable` | No sufficiently relevant chunks were retrieved |
| `unknown` | Answerability could not be determined (e.g. empty KB) |

<Warning>
  Answerability is a heuristic signal, not a guarantee. Always expose a fallback path in your agent when the label is `not_answerable` or `unknown`.
</Warning>
