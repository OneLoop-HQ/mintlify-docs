## Overview

An evaluator binding connects an evaluator to an agent or scenario. Once bound, the evaluator runs automatically against conversations or simulation runs in scope. Marking a binding as `is_critical` means a failure on this evaluator marks the entire session's headline verdict as **failed**.

---

## Create a Binding

```
POST /v1/evaluator-bindings
```

### Request Body

<ParamField body="evaluator_id" type="string (uuid)" required>
  The evaluator to bind.
</ParamField>

<ParamField body="scope" type="string" required>
  `agent` — Runs on all live conversations handled by the agent.
  `scenario` — Runs during simulation runs for the given scenario.
</ParamField>

<ParamField body="agent_id" type="string (uuid)">
  Required when `scope` is `agent`.
</ParamField>

<ParamField body="scenario_id" type="string (uuid)">
  Required when `scope` is `scenario`.
</ParamField>

<ParamField body="is_critical" type="boolean" required>
  When `true`, a failing result on this evaluator causes the session's overall verdict to be `failed`.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/evaluator-bindings \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "evaluator_id": "eval_aa11bb22-0000-0000-0000-000000000060",
    "scope": "agent",
    "agent_id": "a1b2c3d4-0000-0000-0000-000000000001",
    "is_critical": true
  }'
```

```json Response
{
  "id": "bind_cc33dd44-0000-0000-0000-000000000070",
  "evaluator_id": "eval_aa11bb22-0000-0000-0000-000000000060",
  "scope": "agent",
  "agent_id": "a1b2c3d4-0000-0000-0000-000000000001",
  "is_critical": true,
  "created_at": "2024-06-01T11:00:00Z"
}
```
</CodeGroup>

## Update a Binding

```
PATCH /v1/evaluator-bindings/{id}
```

### Request Body

<ParamField body="is_critical" type="boolean">
  Update the criticality flag.
</ParamField>

---

## Delete a Binding

```
DELETE /v1/evaluator-bindings/{id}
```

Removes the binding. The evaluator will no longer run for the associated agent or scenario.

## Eval Runs

Eval runs are the individual execution records of an evaluator against a specific session or simulation. They record the result, score, and any evidence.

### List Eval Runs

```
GET /v1/eval-runs
```

#### Query Parameters

<ParamField query="session_id" type="string">
  Filter to runs from a specific session.
</ParamField>

<ParamField query="evaluator_id" type="string">
  Filter to runs of a specific evaluator.
</ParamField>

<ParamField query="agent_id" type="string">
  Filter to runs involving a specific agent.
</ParamField>

<ParamField query="status" type="string">
  Run status filter.
</ParamField>

<ParamField query="scope" type="string">
  `live`, `simulation`, or `test`.
</ParamField>

<ParamField query="outcome" type="string">
  `passed`, `failed`, `no_verdict`, or `errored`.
</ParamField>

<ParamField query="score_min" type="number">
  Minimum score filter.
</ParamField>

<ParamField query="score_max" type="number">
  Maximum score filter.
</ParamField>

<ParamField query="severity" type="string">
  Filter by evaluator severity.
</ParamField>

<ParamField query="channel" type="string">
  Filter by conversation channel.
</ParamField>

<ParamField query="created_from" type="string (ISO 8601)">
  Filter to runs created after this timestamp.
</ParamField>

<ParamField query="created_to" type="string (ISO 8601)">
  Filter to runs created before this timestamp.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor.
</ParamField>

---

### Get an Eval Run

```
GET /v1/eval-runs/{id}
```

Fetch a single eval run. Use the `include` parameter to embed related data.

#### Query Parameters

<ParamField query="include" type="string">
  Comma-separated list of embedded resources. Options: `session`, `turns`, `recording`.
</ParamField>

---

### Delete an Eval Run

```
DELETE /v1/eval-runs/{id}
```

Only runs in a terminal state (`passed`, `failed`, `errored`) can be deleted.

---

### Retry an Eval Run

```
POST /v1/eval-runs/{id}/retry
```

Re-executes the evaluator against the same session/turns. Useful when the judge model was updated or a transient error occurred.

---

## Get an Eval Score

```
GET /v1/eval-scores/{id}
```

Returns the detailed score record associated with an eval run, including the judge's explanation and raw model output for `model_judge` evaluators.

---

## Re-evaluate a Session

Run all bound evaluators against an existing session again.

### Sessions (v1)

```
POST /v1/sessions/{id}/reevaluate
```

### Conversations (v2)

```
POST /v1/v2/conversations/{id}/reevaluate
```

<Note>
  Re-evaluation enqueues a new set of eval runs and does not delete the previous results. Query `GET /v1/eval-runs?session_id={id}` after re-evaluation to see all run history.
</Note>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/sessions/sess_9f3e2a1b-0000-0000-0000-000000000042/reevaluate \
  -H "x-api-key: <key>"
```

```json Response
{
  "session_id": "sess_9f3e2a1b-0000-0000-0000-000000000042",
  "enqueued_runs": 3,
  "message": "Re-evaluation enqueued. Results will be available shortly."
}
```
</CodeGroup>
