## Overview

Evaluators define quality checks that Feather runs against agent responses during live conversations, simulations, or test runs. There are two kinds:

- **`model_judge`** — An LLM evaluates the response against a natural-language prompt you write.
- **`deterministic`** — A pre-built signal from the Feather signal catalog (e.g. message length, sentiment score, response time) evaluated against a threshold.

Evaluators are reusable across agents and scenarios via [bindings](/api-reference/evaluators/bindings).

---

## Create an Evaluator

```
POST /v1/evaluators
```

### Request Body

<ParamField body="name" type="string" required>
  A short, descriptive name, e.g. `Tone Check` or `Response Completeness`.
</ParamField>

<ParamField body="description" type="string">
  What this evaluator measures and why it matters.
</ParamField>

<ParamField body="owner" type="string">
  User or team responsible for this evaluator.
</ParamField>

<ParamField body="status" type="string">
  `active`, `inactive`, or `archived`. Default: `active`.
</ParamField>

<ParamField body="kind" type="string" required>
  `model_judge` or `deterministic`.
</ParamField>

<ParamField body="config" type="object">
  Evaluator-specific configuration. For `deterministic`, specify `signal_key` and comparison parameters.
</ParamField>

<ParamField body="prompt" type="string">
  Required for `model_judge`. The evaluation instruction given to the judge model. Should describe what to look for and how to score it.
</ParamField>

<ParamField body="threshold" type="number">
  Numeric threshold for pass/fail determination. For `boolean` format, use `1`. For `score` or `numeric`, set the minimum passing value.
</ParamField>

<ParamField body="severity" type="string">
  How serious a failure is: `info`, `low`, `medium`, `high`, or `critical`. Affects the session headline verdict when `is_critical` is set on the binding.
</ParamField>

<ParamField body="judge_model_override" type="string">
  Override the organization-level judge model for this specific evaluator only.
</ParamField>

<ParamField body="format" type="string">
  Output format of the evaluator result:
  - `boolean` — Pass or fail.
  - `enum` — One of a set of labeled categories.
  - `numeric` — A raw number.
  - `score` — A normalized 0–1 score.
</ParamField>

<ParamField body="tags" type="array of strings">
  Labels for organizing and filtering evaluators (e.g. `["safety", "tone", "compliance"]`).
</ParamField>

### Response

Returns an [EvaluatorResponse](#evaluatorresponse).

## Update an Evaluator

```
PATCH /v1/evaluators/{id}
```

Partially updates an evaluator. All fields from the create body are patchable.

---

## Archive an Evaluator

```
POST /v1/evaluators/{id}/archive
```

Sets the evaluator's status to `archived`. Archived evaluators are excluded from new bindings but existing binding history is preserved.

---

## Deterministic Signal Catalog

```
GET /v1/evals/signals
```

Returns the catalog of built-in deterministic signals available for `deterministic` evaluators.

### Response

<ResponseField name="signals" type="array of objects">
  Each signal entry:

  - `key` (string) — The signal identifier to use as `config.signal_key`.
  - `label` (string) — Human-readable signal name.
  - `unit` (string) — Measurement unit (e.g. `ms`, `characters`, `score`).
  - `format` (string) — Output format: `boolean`, `numeric`, or `score`.
  - `aggregations` (array of strings) — Available aggregation methods: `avg`, `p50`, `p95`, `count`.
  - `channels` (array of strings) — Channels where this signal is available.
</ResponseField>

## Examples

### Model Judge Evaluator

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/evaluators \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Professional Tone Check",
    "description": "Checks that the agent response is professional and not dismissive.",
    "kind": "model_judge",
    "format": "boolean",
    "prompt": "Does the following agent response maintain a professional and empathetic tone? Answer YES or NO, then briefly explain.",
    "threshold": 1,
    "severity": "medium",
    "tags": ["tone", "cx-quality"]
  }'
```

```json Response
{
  "id": "eval_aa11bb22-0000-0000-0000-000000000060",
  "name": "Professional Tone Check",
  "kind": "model_judge",
  "format": "boolean",
  "threshold": 1,
  "severity": "medium",
  "status": "active",
  "tags": ["tone", "cx-quality"],
  "created_at": "2024-06-01T10:00:00Z"
}
```
</CodeGroup>

### Deterministic Evaluator

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/evaluators \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Response Length Guard",
    "description": "Fails if the agent response exceeds 500 characters.",
    "kind": "deterministic",
    "format": "boolean",
    "config": {
      "signal_key": "response_character_count",
      "operator": "lte",
      "value": 500
    },
    "threshold": 1,
    "severity": "low",
    "tags": ["length"]
  }'
```
</CodeGroup>
