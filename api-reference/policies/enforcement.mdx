## Overview

This page covers the supporting endpoints that help you author and audit policies:

- **Capabilities** — discover which enforcement points, check types, and resolution methods are available in your plan.
- **Expression Schema** — explore what fields and operators the expression DSL can reference at each point.
- **Tool Schema** — fetch the input and output schemas for a specific tool to write precise pre/post-tool policies.
- **Evaluations** — query the audit log of every policy check that ran.

---

## Get Policy Capabilities

```
GET /v1/policies/capabilities
```

Returns the complete authoring menu for your organization — which enforcement points are enabled, which check types are available, and which resolution modes are supported.

### Response

<ResponseField name="points" type="array of strings">
  Enforcement points available: e.g. `["input", "pre_tool", "post_tool", "agent_response"]`.
</ResponseField>

<ResponseField name="check_types" type="array of strings">
  Check types available: `["expression", "llm_judge"]`.
</ResponseField>

<ResponseField name="resolutions" type="array of strings">
  Resolution modes available for the `llm_judge` check type.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/policies/capabilities \
  -H "x-api-key: <key>"
```

```json Response
{
  "points": ["input", "pre_tool", "post_tool", "agent_response"],
  "check_types": ["expression", "llm_judge"],
  "resolutions": ["boolean", "score"]
}
```
</CodeGroup>

## Get Expression Schema

```
GET /v1/policies/expression-schema
```

Returns the full reference for the Feather expression DSL — all predicates, operators, common fields, and the content paths available at each enforcement point.

### Response

<ResponseField name="predicates" type="array of strings">
  Functions usable in expressions, e.g. `contains`, `contains_any`, `matches_regex`, `starts_with`, `equals`, `length_gt`.
</ResponseField>

<ResponseField name="operators" type="array of strings">
  Logical and comparison operators: `AND`, `OR`, `NOT`, `==`, `!=`, `>`, `<`.
</ResponseField>

<ResponseField name="common_fields" type="array of strings">
  Fields available at all enforcement points: e.g. `session.channel`, `session.language`, `agent.id`.
</ResponseField>

<ResponseField name="points" type="array of objects">
  Per-enforcement-point field availability. Each entry contains:
  - `enforcement_point` (string) — The pipeline stage.
  - `content_paths` (array of strings) — The dotted-path fields accessible in expressions at this point.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/policies/expression-schema \
  -H "x-api-key: <key>"
```

```json Response
{
  "predicates": ["contains", "contains_any", "matches_regex", "starts_with", "equals", "length_gt"],
  "operators": ["AND", "OR", "NOT"],
  "common_fields": ["session.channel", "session.language", "agent.id"],
  "points": [
    {
      "enforcement_point": "input",
      "content_paths": ["input.text", "input.media_urls"]
    },
    {
      "enforcement_point": "agent_response",
      "content_paths": ["response.text", "response.citations", "response.tool_calls"]
    },
    {
      "enforcement_point": "pre_tool",
      "content_paths": ["tool.name", "tool.input"]
    },
    {
      "enforcement_point": "post_tool",
      "content_paths": ["tool.name", "tool.output", "tool.error"]
    }
  ]
}
```
</CodeGroup>

### Expression Examples

```
# Block responses mentioning refunds
response.text contains("refund")

# Block if message is very long and contains a link
input.text length_gt(500) AND input.text contains("http")

# Scope to a specific tool by name
tool.name equals("send_email") AND tool.input contains("unsubscribe")
```

## Get Tool Schema

```
GET /v1/policies/tool-schema?tool=api:<uuid>
```

Fetches the input and output JSON schemas for a specific tool, so you can write precise `pre_tool` and `post_tool` expression policies that reference real tool field names.

### Query Parameters

<ParamField query="tool" type="string" required>
  Tool identifier in the format `api:<tool_uuid>`.
</ParamField>

### Response

<ResponseField name="tool_id" type="string">
  The resolved tool identifier.
</ResponseField>

<ResponseField name="input_schema" type="object">
  JSON Schema describing the tool's input parameters.
</ResponseField>

<ResponseField name="output_schema" type="object">
  JSON Schema describing the tool's output fields.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/policies/tool-schema?tool=api:b2c3d4e5-0000-0000-0000-000000000050" \
  -H "x-api-key: <key>"
```

```json Response
{
  "tool_id": "api:b2c3d4e5-0000-0000-0000-000000000050",
  "input_schema": {
    "type": "object",
    "properties": {
      "customer_id": { "type": "string" },
      "refund_amount": { "type": "number" }
    },
    "required": ["customer_id", "refund_amount"]
  },
  "output_schema": {
    "type": "object",
    "properties": {
      "status": { "type": "string", "enum": ["approved", "declined"] },
      "transaction_id": { "type": "string" }
    }
  }
}
```
</CodeGroup>

## Policy Evaluations Audit Log

```
GET /v1/policies/evaluations
```

Returns a paginated log of every policy evaluation event. Use this to audit what policies ran, what they found, and what actions were taken.

### Query Parameters

<ParamField query="conversation_id" type="string">
  Filter to evaluations from a specific conversation.
</ParamField>

<ParamField query="policy_id" type="string">
  Filter to a specific policy.
</ParamField>

<ParamField query="result" type="string">
  Filter by evaluation outcome: `pass`, `fail`, `error`, or `skipped`.
</ParamField>

<ParamField query="enforcement_point" type="string">
  Filter by pipeline stage: `input`, `pre_tool`, `post_tool`, or `agent_response`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor.
</ParamField>

### Response: PolicyEvaluationResponse

<ResponseField name="result" type="string">
  Evaluation outcome: `pass`, `fail`, `error`, or `skipped`.
</ResponseField>

<ResponseField name="action_taken" type="string">
  The action that was actually applied (only in `enforce` mode when `result=fail`).
</ResponseField>

<ResponseField name="would_be_action" type="string">
  The action that would have been applied in `enforce` mode (present for `monitor` mode records).
</ResponseField>

<ResponseField name="violation_detail" type="object">
  Details about the triggered violation. For expression checks, includes the matched expression and matched content. For `llm_judge` checks, includes the judge's reasoning.
</ResponseField>

<ResponseField name="latency_ms" type="integer">
  Time taken to evaluate the policy in milliseconds.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl "https://api-sandbox.featherhq.com/v1/policies/evaluations?result=fail&enforcement_point=agent_response" \
  -H "x-api-key: <key>"
```
