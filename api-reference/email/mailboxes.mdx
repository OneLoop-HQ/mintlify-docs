## Overview

Feather connects to Gmail and Microsoft 365 mailboxes using OAuth 2.0. Once a mailbox is authorized, you can bind it to a specific agent or team, configure a default HTML signature, and manage the connection lifecycle. No passwords or SMTP credentials are stored.

---

## Connect a Mailbox

```
POST /v1/email/accounts/connect
```

Initiates the OAuth flow for a Gmail or Microsoft 365 mailbox. Returns an authorization URL that you redirect the user to. After the user grants permission, Feather handles the token exchange automatically.

<Note>
  You must supply at least one of `agent_id`, `team_id`, or leave both blank to connect an unbound mailbox you can bind later with `PATCH /v1/email/accounts/{id}`.
</Note>

### Request Body

<ParamField body="agent_id" type="string (uuid)">
  Bind the mailbox to this agent after authorization completes.
</ParamField>

<ParamField body="agent_revision_id" type="string (uuid)">
  Pin the binding to a specific agent revision. If omitted, the binding tracks the agent's active revision.
</ParamField>

<ParamField body="team_id" type="string (uuid)">
  Bind the mailbox to this team after authorization completes.
</ParamField>

<ParamField body="team_revision_id" type="string (uuid)">
  Pin the binding to a specific team revision.
</ParamField>

<ParamField body="login_hint" type="string">
  Pre-fills the email address in the OAuth consent screen. Useful when you already know which account the user intends to connect.
</ParamField>

### Response

<ResponseField name="authorize_url" type="string">
  The full OAuth authorization URL. Redirect the user's browser to this URL.
</ResponseField>

<ResponseField name="state" type="string">
  An opaque state token tied to this authorization request. Feather validates this on the callback — do not modify it.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.featherhq.com/v1/email/accounts/connect \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "a1b2c3d4-0000-0000-0000-000000000001",
    "login_hint": "support@acme.com"
  }'
```

```json Response
{
  "authorize_url": "https://accounts.google.com/o/oauth2/v2/auth?client_id=...&state=feather_state_abc123",
  "state": "feather_state_abc123"
}
```
</CodeGroup>

## List Mailboxes

```
GET /v1/email/accounts
```

Returns all connected mailboxes for your organization.

### Response

<ResponseField name="accounts" type="array of MailboxAccountResponse">
  List of connected mailbox objects. See [MailboxAccountResponse](#mailboxaccountresponse) below.
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl https://api-sandbox.featherhq.com/v1/email/accounts \
  -H "x-api-key: <key>"
```

```json Response
{
  "accounts": [
    {
      "id": "mbox_1a2b3c4d-0000-0000-0000-000000000010",
      "address": "support@acme.com",
      "provider": "gmail",
      "status": "active",
      "capabilities": ["send", "receive", "drafts"],
      "default_signature_html": "<p>Best regards,<br/>Acme Support</p>"
    }
  ]
}
```
</CodeGroup>

## Get a Mailbox

GET /v1/email/accounts/{id}

Returns a single mailbox by ID.

## Update a Mailbox

```
PATCH /v1/email/accounts/{id}
```

Updates the signature or rebinds the mailbox to a different agent or team — without requiring a new OAuth flow.

### Request Body

<ParamField body="default_signature_html" type="string">
  Raw HTML inserted at the bottom of every email composed from this mailbox. Pass an empty string to remove the signature.
</ParamField>

<ParamField body="agent_id" type="string (uuid)">
  Rebind the mailbox to a different agent. Clears any existing team binding.
</ParamField>

<ParamField body="team_id" type="string (uuid)">
  Rebind the mailbox to a different team. Clears any existing agent binding.
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X PATCH https://api-sandbox.featherhq.com/v1/email/accounts/mbox_1a2b3c4d-0000-0000-0000-000000000010 \
  -H "x-api-key: <key>" \
  -H "Content-Type: application/json" \
  -d '{
    "default_signature_html": "<p>Thanks,<br/>Acme Support Team</p>",
    "agent_id": "a1b2c3d4-0000-0000-0000-000000000002"
  }'
```
</CodeGroup>

## Delete a Mailbox

```
DELETE /v1/email/accounts/{id}
```

Revokes Feather's OAuth access to the mailbox and retires the account record. Existing sessions and draft history are preserved, but new messages will no longer be received or sent from this address.

<Warning>
  Deletion is irreversible. The user must go through the OAuth flow again to reconnect the same address.
</Warning>

### Example

<CodeGroup>
```bash cURL
curl -X DELETE https://api-sandbox.featherhq.com/v1/email/accounts/mbox_1a2b3c4d-0000-0000-0000-000000000010 \
  -H "x-api-key: <key>"
```
</CodeGroup>

## MailboxAccountResponse

<ResponseField name="id" type="string">
  Unique mailbox identifier.
</ResponseField>

<ResponseField name="address" type="string">
  The email address associated with this mailbox (e.g. `support@acme.com`).
</ResponseField>

<ResponseField name="provider" type="string">
  OAuth provider. One of `gmail` or `microsoft`.
</ResponseField>

<ResponseField name="status" type="string">
  Current connection state:

  - `active` — Connected and healthy.
  - `pending` — OAuth flow initiated but not yet completed.
  - `error` — An error occurred during token refresh or message sync.
  - `needs_reauth` — The OAuth token has been revoked or expired. The user must reconnect.
</ResponseField>

<ResponseField name="capabilities" type="array of strings">
  Actions this mailbox supports. Common values: `send`, `receive`, `drafts`.
</ResponseField>

<ResponseField name="default_signature_html" type="string">
  HTML email signature appended to outbound messages. May be `null` if no signature is configured.
</ResponseField>
